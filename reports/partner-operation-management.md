# 1. 文書概要

## 1.1 目的

本設計書は、協力パートナーの **登録・審査・反社チェック・電子契約締結後の実運用** を標準化し、Builder 管理画面で継続管理できるデータ構造・ルール・画面構成を定義する。

- **案件紹介可否** の判定ロジックを明確化する
- **保険・労災・インボイス・許可・資格** の期限管理を統一する
- **契約停止・解除** および **履歴管理** のルールを整理する
- 既存3文書（審査基準・反社チェック・電子契約運用）との整合を保ち、Builder への実装落とし込みを可能にする

## 1.2 対象区分

| 区分 | 説明 |
|------|------|
| 法人 | 株式会社・合同会社等 |
| 個人事業主 | 開業届提出済みの個人事業 |
| 一人親方 | 個人で現場作業を行う事業者 |
| フリーランス | 個人でサービスを提供する事業者 |

## 1.3 対象範囲（運用管理の対象パートナー）

| 区分 | 審査ステータス | 契約状態 | 運用管理の扱い |
|------|---------------|----------|---------------|
| **契約済み協力パートナー** | `contracted` | `active` | 案件紹介・更新・評価の主対象 |
| **承認済み未契約者** | `approved` | — | 契約送付・督促管理（電子契約運用フロー） |
| **審査中・保留** | `pending` / `hold` | — | 審査・反社チェック管理 |
| **停止中パートナー** | `contracted` | `suspended` | 停止理由管理・更新待ち・進行案件の個別判断 |
| **解除済みパートナー** | `contracted` または `rejected` | `terminated` | 履歴参照・再登録可否の管理 |

## 1.4 ライフサイクル全体像

```
登録申請 → 審査 → 反社チェック → 承認 → 電子契約 → 契約済み
                                              ↓
                                    案件紹介・稼働・評価
                                              ↓
                              更新管理（保険・労災・インボイス等）
                                              ↓
                              停止（一時） / 解除（終了）
```

本設計書は **承認以降の継続運用** を中心に定義する。審査・反社・契約の詳細は各関連文書を参照する。

# 3. 案件紹介可否ルール

## 3.1 案件紹介可能条件（すべて必須）

以下 **7条件をすべて満たす場合のみ** `referral_eligible = true` とする。
**1つでも満たさない場合は紹介不可** とする。

| # | 条件 | 判定基準 |
|---|------|----------|
| 1 | **ステータスが契約済み** | 審査ステータス = `contracted` |
| 2 | **契約状態が有効** | `contract_validity` = `active` |
| 3 | **反社判定が問題なし** | 反社判定 = `clear` |
| 4 | **インボイス有効** | インボイス状態 = `valid` |
| 5 | **保険有効** | 保険期限 ≧ 当日、対人≧1億円・対物≧5,000万円 |
| 6 | **労災有効** | 区分・業種に応じた労災加入が有効 |
| 7 | **停止中ではない** | `contract_validity` ≠ `suspended` かつ 停止フラグなし |

### 判定ロジック（疑似コード）

```
referral_eligible =
  status == 'contracted'
  AND contract_validity == 'active'
  AND antisocial_status == 'clear'
  AND invoice_status == 'valid'
  AND insurance_expiry >= today
  AND workers_comp_valid == true
  AND NOT is_suspended
```

## 3.2 紹介不可理由コード

紹介不可時は、**最優先の理由コード1件** を `referral_block_code` に記録する（複数該当時は下表の優先順位で決定）。

| 優先 | コード | 理由 | 典型対応 |
|------|--------|------|----------|
| 1 | **N07** | 契約解除済み | 再登録審査（条件付き） |
| 2 | **N06** | 反社要確認 / 反社該当 | 保留（H09）または否認（R01） |
| 3 | **N02** | 契約停止中 | 停止解除・更新完了まで待機 |
| 4 | **N05** | インボイス無効 | 再登録確認・停止（S08） |
| 5 | **N03** | 保険期限切れ | 証券再提出（S02） |
| 6 | **N04** | 労災期限切れ / 未加入 | 証明再提出（S03） |
| 7 | **N08** | 許可証・資格期限切れ | 該当業種の紹介のみ停止（S09） |
| 8 | **N09** | 稼働評価「停止検討」 | 運営管理者判断で紹介停止 |
| 9 | **N01** | 未契約（承認済み未締結含む） | 電子契約完了を待つ |
| 10 | **N10** | 審査未承認（pending/hold/rejected） | 審査完了を待つ |
| 11 | **N11** | 対応エリア外 | 案件エリアと不一致 |
| 12 | **N12** | 業種不一致 | 案件業種と不一致 |

**ユーザー指定コードとの対応**: N01〜N07 は指定どおり。運用上必要な N08〜N12 を追加定義。

## 3.3 案件紹介可否の表示（Builder）

| 表示 | 条件 |
|------|------|
| 🟢 紹介可 | `referral_eligible = true` |
| 🟡 紹介不可（一時） | N02〜N06, N08（更新・確認で復帰可能） |
| 🔴 紹介不可（恒久） | N07, N06（該当時）, N10（rejected） |

# 4. 案件紹介管理

## 4.1 紹介レコード（1紹介 = 1レコード）

| 項目名 | フィールドキー | 型 | 必須 | 説明 |
|--------|---------------|-----|------|------|
| 紹介ID | `referral_id` | string | ○ | REF-YYYY-NNNNN |
| パートナーID | `partner_id` | string | ○ | 台帳へのFK |
| 紹介日 | `referred_at` | date | ○ | 紹介実施日 |
| 案件ID | `project_id` | string | ○ | 案件・依頼の識別子 |
| 案件種別 | `project_type` | string | ○ | 内装・電気・IT 等 |
| 対応エリア | `project_area` | string | ○ | 案件のエリア |
| 紹介担当者 | `referrer` | string | ○ | 紹介を実施した運営担当 |
| 紹介結果 | `referral_result` | enum | ○ | 下表参照 |
| 受注有無 | `order_received` | boolean | — | 受注したか |
| 辞退理由 | `decline_reason` | string | — | 辞退時の理由 |
| 完了有無 | `completed` | boolean | — | 案件完了したか |
| トラブル有無 | `has_trouble` | boolean | — | トラブル発生有無 |
| 最終紹介日 | `last_referral_at` | date | — | 当該パートナーへの直近紹介日（集計用） |
| 備考 | `notes` | text | — | |

### 紹介結果（referral_result）

| 値 | 意味 |
|----|------|
| `pending` | 紹介済み・パートナー回答待ち |
| `accepted` | パートナー受諾 |
| `declined` | パートナー辞退 |
| `no_response` | 無回答 |
| `matched` | マッチング成立（発注先決定） |
| `lost` | 他社受注・不成立 |
| `completed` | 案件完了 |
| `cancelled` | 案件キャンセル |

## 4.2 パートナー別集計項目（台帳にキャッシュ）

| 集計項目 | フィールドキー | 算出方法 |
|----------|---------------|----------|
| 紹介回数 | `referral_count` | 紹介レコードの件数 |
| 受注回数 | `order_count` | `order_received = true` の件数 |
| 受注率 | `order_rate` | 受注回数 / 紹介回数（%） |
| 完了件数 | `completed_count` | `completed = true` の件数 |
| トラブル件数 | `trouble_count` | `has_trouble = true` の件数 + トラブル管理の件数 |
| 最終稼働日 | `last_active_at` | 直近の案件完了日または紹介日 |

## 4.3 紹介実施前チェック

案件紹介操作前に、第3章の `referral_eligible` を再計算し、false の場合は紹介操作をブロックする（将来は Builder UI で制御）。

# 5. 稼働評価

## 5.1 評価項目（各5段階: 5=優秀 〜 1=要改善）

| # | 評価項目 | 評価観点 |
|---|----------|----------|
| 1 | **連絡速度** | 問い合わせ・紹介への返信速度 |
| 2 | **見積対応** | 見積提出の速さ・内容の明確さ |
| 3 | **施工品質** | 仕上がり・技術力（サービス品質） |
| 4 | **顧客対応** | 施主・利用者への対応態度 |
| 5 | **納期遵守** | 工期・納期の遵守 |
| 6 | **報告品質** | 進捗報告・完了報告の適時性・内容 |
| 7 | **トラブル率** | トラブル・クレームの発生頻度・重大度 |

## 5.2 評価ランク

総合スコア（7項目の平均または加重平均）に基づき、以下のランクを付与する。

| ランク | スコア目安 | 案件紹介方針 |
|--------|-----------|-------------|
| **S** | 4.5以上 | **優先紹介**。高単価・重要案件を優先的にアサイン |
| **A** | 4.0〜4.4 | **積極紹介**。通常案件を問題なく紹介 |
| **B** | 3.0〜3.9 | **通常紹介**。案件内容に応じて紹介継続 |
| **C** | 2.0〜2.9 | **慎重紹介**。単純・小規模案件に限定。改善指導を実施 |
| **停止検討** | 2.0未満、または重大トラブル | **新規紹介停止**（N09）。運営管理者が停止・解除を判断 |

## 5.3 評価の更新タイミング

| タイミング | 内容 |
|-----------|------|
| 案件完了時 | 担当者が簡易評価（任意） |
| 月次 | 運営が直近の紹介・トラブルを踏まえてランク見直し |
| 重大トラブル発生時 | 即時「停止検討」へ降格検討 |
| 年次 | 正式な総合評価の見直し |

## 5.4 評価記録項目

| 項目 | 説明 |
|------|------|
| 評価日 | 評価実施日 |
| 評価者 | 担当者名 |
| 評価期間 | 対象期間（例: 2026年Q2） |
| 各項目スコア | 1〜5 |
| 総合ランク | S / A / B / C / 停止検討 |
| コメント | 改善点・特記事項 |

# 6. 更新期限管理

## 6.1 管理対象と台帳フィールド

| 管理対象 | フィールドキー | 確認頻度 |
|----------|---------------|----------|
| 保険期限 | `insurance_expiry` | 年次 + 期限前アラート |
| 労災期限 | `workers_comp_expiry` | 年次 + 期限前アラート |
| 建設業許可期限 | `license_expiry` | 該当業種・年次 |
| 資格期限 | `qualification_expiry` | 該当業種・年次（複数可は JSON） |
| インボイス状態 | `invoice_status` | **月次**照合推奨 |
| 契約更新期限 | `contract_renewal_date` | 年次（契約条項に基づく） |

## 6.2 通知タイミング

| タイミング | アクション | 担当 |
|-----------|-----------|------|
| **60日前** | 第1回更新依頼メール | 契約担当 |
| **30日前** | 第2回更新依頼メール | 契約担当 |
| **14日前** | 第3回更新依頼（メール + 電話推奨） | 契約担当 |
| **期限当日** | 案件紹介 **即時停止**。`referral_eligible = false` | システム / 担当者 |
| **期限切れ後** | 契約状態を `suspended` に変更検討 | 契約担当 |

## 6.3 期限切れ時の処理フロー

```
期限切れ検知
    ↓
① 案件紹介停止（referral_eligible = false、N03/N04/N05等を設定）
    ↓
② 契約状態を「停止（suspended）」に変更（保険・労災・インボイスの場合）
    │  ステータスは contracted を維持
    ↓
③ パートナーへ再提出依頼
    ↓
④ 再提出・確認完了
    ├→ OK：期限日更新、active 復帰、紹介再開
    └→ 未対応（14日）：停止維持
         └→ 未対応（30日）：解除（terminated）検討
```

## 6.4 更新期限ダッシュボード（Builder）

| 表示区分 | 条件 |
|----------|------|
| 🔴 期限切れ | 期限 < 当日 |
| 🟡 14日以内 | 当日 ≦ 期限 ≦ 14日後 |
| 🟢 30日以内 | 14日 < 期限 ≦ 30日後 |
| ⚪ 60日以内 | 30日 < 期限 ≦ 60日後 |

# 7. 停止管理

## 7.1 停止事由（電子契約運用 S01〜S10 と整合）

| コード | 停止事由 | 審査ステータス | 契約状態 | 紹介 |
|--------|----------|---------------|----------|------|
| S02 | 保険失効 | `contracted` 維持 | `suspended` | 不可（N03） |
| S03 | 労災失効 | 同上 | `suspended` | 不可（N04） |
| S08 | インボイス無効 | 同上 | `suspended` | 不可（N05） |
| S09 | 許可・資格失効 | 同上 | `suspended` または業種限定停止 | 不可（N08） |
| H09相当 | 反社要確認 | `hold` または `contracted` | `suspended` 検討 | 不可（N06） |
| S05 | 契約違反 | `contracted` | `suspended` | 不可（N02） |
| S06 | 顧客抜き疑い | 同上 | `suspended` | 不可（N02） |
| S07 | SNS違反 | 同上 | `suspended` | 不可（N02） |
| S10 | 重大クレーム | 同上 | `suspended` | 不可（N02） |
| S07相当 | 連絡不能 | 同上 | `suspended` | 不可（N02） |
| S01 | 虚偽申告疑い | 調査中 | `suspended` | 不可 → 解除検討 |
| S04 | 反社判明 | → `rejected` または `terminated` | `terminated` | 不可（N06/N07） |

## 7.2 停止中の扱い

| 項目 | ルール |
|------|--------|
| **新規案件紹介** | **不可**（必須） |
| **進行中案件** | **個別判断**。安全性・契約上の義務を優先し、運営管理者が継続・中止を決定 |
| **Builder記録** | **停止日**（`suspended_at`）、**停止事由コード**（`suspension_code`）、**停止担当者**、**備考** を必須記録 |
| **停止解除** | 事由解消 + 書類再確認後、`active` に復帰 |

## 7.3 停止管理レコード

| 項目 | 説明 |
|------|------|
| 停止ID | `SUS-YYYY-NNNN` |
| パートナーID | 台帳FK |
| 停止日 | 停止開始日 |
| 停止事由コード | S02〜S10 等 |
| 停止担当者 | 実施者 |
| 解除日 | 解除時（null = 停止中） |
| 解除担当者 | 解除実施者 |
| 備考 | 経緯・判断根拠 |

# 8. 契約解除管理

## 8.1 解除事由

| コード | 解除事由 | 再登録可否 | 審査基準との対応 |
|--------|----------|-----------|-----------------|
| T01 | 反社判明 | **不可** | R01 |
| T02 | 虚偽申告 | **不可** | R02/R10 |
| T03 | 顧客抜き | **不可** | S06 |
| T04 | 重大施工不良 | **要審議** | S10 |
| T05 | 保険未更新（長期） | **可**（更新後） | S02 |
| T06 | 労災未更新（長期） | **可**（更新後） | S03 |
| T07 | 連絡不能 | **可**（6ヶ月後） | R07 |
| T08 | 契約違反 | **要審議** | S05 |
| T09 | パートナー申出 | **可** | — |
| T10 | 当社都合 | **要審議** | — |

## 8.2 解除管理項目

| 項目名 | フィールドキー | 必須 | 説明 |
|--------|---------------|------|------|
| 解除日 | `terminated_at` | ○ | 契約解除日 |
| 解除理由コード | `termination_code` | ○ | T01〜T10 |
| 解除担当者 | `terminated_by` | ○ | 実施者 |
| 証跡 | `termination_evidence` | 推奨 | 通知メール・調査記録の保管先 |
| 再登録可否 | `re_registration_allowed` | ○ | true / false / conditional |
| 備考 | `termination_notes` | ○ | 判断根拠・経緯 |

## 8.3 解除時のシステム更新

| フィールド | 更新値 |
|-----------|--------|
| `contract_validity` | `terminated` |
| `referral_eligible` | `false` |
| `referral_block_code` | `N07` |

# 9. トラブル・クレーム管理

## 9.1 管理項目

| 項目名 | フィールドキー | 型 | 必須 | 説明 |
|--------|---------------|-----|------|------|
| トラブルID | `trouble_id` | string | ○ | TRB-YYYY-NNNN |
| パートナーID | `partner_id` | string | ○ | 台帳FK |
| 発生日 | `occurred_at` | date | ○ | トラブル発生日 |
| 案件ID | `project_id` | string | — | 関連案件 |
| 内容 | `description` | text | ○ | トラブル内容 |
| 重大度 | `severity` | enum | ○ | 低 / 中 / 高 / 重大 |
| 対応状況 | `status` | enum | ○ | 受付 / 調査中 / 対応中 / 完了 |
| 責任区分 | `responsibility` | enum | — | パートナー / 当社 / 双方 / 不明 |
| 再発防止策 | `prevention` | text | — | 再発防止の内容 |
| 対応完了日 | `resolved_at` | date | — | クローズ日 |
| 備考 | `notes` | text | — | |

## 9.2 重大度の定義

| 重大度 | 定義 | 対応 |
|--------|------|------|
| **低** | 軽微な連絡遅延・仕上がりの小瑕疵 | 記録のみ。パートナーへフィードバック |
| **中** | 顧客クレーム・納期遅延・手直し要 | 記録 + 改善指導。評価へ反映 |
| **高** | 重大な品質不良・顧客被害・法的リスクの萌芽 | **停止検討**。運営管理者へエスカレーション |
| **重大** | 安全事故・重大な法的違反・組織的不正 | **即時停止または解除検討**（S10/T01〜T04） |

## 9.3 トラブルと稼働評価・停止の連動

| 条件 | アクション |
|------|-----------|
| 重大度「高」が1件 | 稼働評価を「停止検討」へ降格検討 |
| 重大度「重大」が1件 | 即時 `suspended`、解除検討 |
| 重大度「中」が年間3件以上 | ランク降格・改善面談 |

# 11. ステータス連動

以下は、イベント発生時の **フィールド自動更新ルール** を定義する（将来は Edge Function / DB トリガーで実装）。

## 11.1 審査・契約フェーズ

| イベント | 審査ステータス | 契約状態 | 案件紹介可否 | 備考 |
|----------|---------------|----------|-------------|------|
| 登録申請受付 | `pending` | — | 不可 N10 | 自動 |
| 保留設定 | `hold` | — | 不可 N10 | Hコードを記録 |
| 審査承認 | `approved` | — | 不可 N01 | 承認日を記録 |
| 否認 | `rejected` | — | 不可 N10 | Rコードを記録 |
| 電子契約完了 | `contracted` | `active` | **可**（他条件OK時） | 契約日を記録 |

## 11.2 運用フェーズ

| イベント | 審査ステータス | 契約状態 | 案件紹介可否 | 紹介不可コード |
|----------|---------------|----------|-------------|---------------|
| 契約状態 active かつ全条件OK | `contracted` | `active` | **可** | — |
| 保険失効 | `contracted` | `suspended` | 不可 | N03 |
| 労災失効 | `contracted` | `suspended` | 不可 | N04 |
| インボイス無効 | `contracted` | `suspended` | 不可 | N05 |
| 反社要確認 | `hold` または `contracted` | `suspended` 検討 | 不可 | N06 |
| 反社該当 | `rejected` | `terminated` | 不可 | N06 |
| 停止（その他） | `contracted` | `suspended` | 不可 | N02 |
| 契約解除 | `contracted` | `terminated` | 不可 | N07 |
| 停止解除・更新完了 | `contracted` | `active` | 条件付き可 | 再計算 |

## 11.3 連動の優先順位

複数の問題が同時に存在する場合、`referral_block_code` は **第3.2章の優先順位** に従い1件を表示する。台帳の `notes` には複数理由をすべて記録する。
