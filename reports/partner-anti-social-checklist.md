# 1. 文書概要

## 1.1 目的

本チェックシートは、協力パートナー審査における **反社会的勢力（反社）確認** を標準化し、以下を実現する。

- 審査担当者による判断のばらつきを防ぐ
- 反社確認の実施内容・証跡・判定結果を記録可能にする
- Builder 管理画面・審査基準・電子契約運用と連携できる構成を提供する
- 将来的な外部反社チェックサービス / API 導入時にも拡張しやすい枠組みとする

## 1.2 対象

本チェックシートの確認対象は、協力パートナー登録申請を行った事業体およびその関係者とする。

| 区分 | 対象 |
|------|------|
| **法人** | 申請法人およびその代表者・役員・実質支配者（判明する範囲） |
| **個人事業主** | 申請者本人および屋号で営む事業 |
| **一人親方** | 申請者本人 |
| **フリーランス** | 申請者本人（屋号・個人名で活動する場合を含む） |

## 1.3 利用タイミング

| タイミング | 内容 |
|-----------|------|
| **初回審査時** | 登録申請受付後、承認判断の前に必ず実施 |
| **保留解除時** | 反社関連で「要確認」となった場合、追加確認後に再判定 |
| **契約更新・重要変更時** | 代表者変更・商号変更等、将来運用で再確認が必要な場合（要運営判断） |
| **電子契約前** | 承認後・電子契約案内前の最終確認として実施（推奨） |

**原則：反社チェックが「問題なし」で完了するまで、ステータスを「承認」にしてはならない。**

## 1.4 審査フロー上の位置付け

本チェックシートは、審査基準第7章の標準フローにおいて **一次確認の直後・保険・労災等の確認と並行または先行** して実施する。

```
登録申請（申請者）
    ↓
一次確認（審査担当）
    │  申請内容の欠損・明らかな矛盾のチェック
    ↓
★ 反社チェック（本チェックシート） ★  ← ここで実施
    │  公開情報調査・SNS・行政処分・自己申告の確認
    │  → 該当：否認（R01）
    │  → 要確認：保留（H09）
    │  → 問題なし：次工程へ
    ↓
保険確認 / 労災確認 / インボイス確認 / 実績確認 等
    ↓
承認判断（審査基準 第9章チェックリスト）
    ↓
電子契約
    ↓
契約済み・案件紹介開始
```

# 2. 判定区分

反社チェックの判定は **3段階** とする。Builder 管理画面の「反社」フィールドおよび本チェックシートの判定欄で共通利用する。

| 判定 | 内部コード | 定義 | 審査への影響 |
|------|-----------|------|-------------|
| **問題なし** | `clear` | 反社・暴力団関係・重大違法関与の情報がなく、合理的な疑いもない | 反社チェック完了（審査基準チェックリスト項目1 = YES）。次工程へ進行可 |
| **要確認** | `review` | 情報不足・同姓同名・ネガティブ情報の事実不明等により、現時点では断定できない | ステータス「保留」（H09）。追加確認後に再判定 |
| **該当** | `hit` | 反社会的勢力との関与が合理的に認められる、または該当の蓋然性が極めて高い | ステータス「否認」（R01）。原則再申請不可 |

### 2.1 判定別の詳細定義

#### 問題なし（clear）

- 確認対象について、反社DB・公開情報・SNS・行政処分等を調査した結果、反社関連情報が **確認されない**
- ネガティブ情報があっても、**別人・別法人** であることが合理的に判断できる
- 申請者の自己申告と調査結果に **矛盾がない**

#### 要確認（review）

- 確認に必要な情報が **不足** している（HPなし・連絡不能・実在不明等）
- 検索結果が **同姓同名** で申請者との同一性が判断できない
- ネガティブ情報があるが **事実関係が不明** で断定できない
- SNSが **非公開・削除済み** で確認が不十分
- 将来導入予定の外部APIが **要確認フラグ** を返した場合

#### 該当（hit）

- 反社認定情報・暴力団関係情報が **確認された**
- 犯罪組織への関与・マネーロンダリング関与が **合理的に認められる**
- 継続的な重大違法行為・組織的犯罪への関与が **確認された**
- 申請時の自己申告（反社でない）と調査結果が **明らかに矛盾** する（R10 も併記可）

### 2.2 判定と審査ステータスの対応

| 反社判定 | 審査ステータス | 理由コード |
|----------|---------------|-----------|
| 問題なし | 審査継続 → 他条件クリア後に承認可 | — |
| 要確認 | **保留** | H09（反社チェック追加確認中） |
| 該当 | **否認** | R01（反社該当） |

# 3. 確認対象

区分に応じて、以下の者・情報を確認対象とする。

## 3.1 法人の場合

| 確認対象 | 確認内容 | 優先度 |
|----------|----------|--------|
| **会社名** | 登記上の名称・申告名称の一致 | 必須 |
| **商号** | 屋号・ブランド名・通称の有無 | 必須 |
| **代表者** | 代表取締役・代表社員等の氏名 | 必須 |
| **役員** | 取締役・監査役等（判明する範囲） | 推奨 |
| **実質支配者** | 議決権・出資・支配関係から判明する者（判明する範囲） | 推奨 |

**補足**

- 法人番号公表サイト・登記情報で代表者・本店所在地を照合する
- 実質支配者が判明しない場合は、代表者を中心に確認し、備考に「実質支配者：確認範囲外」と記録する

## 3.2 個人（個人事業主・一人親方・フリーランス）の場合

| 確認対象 | 確認内容 | 優先度 |
|----------|----------|--------|
| **氏名** | 申告氏名・契約名義 | 必須 |
| **屋号** | 屋号・事業名（ある場合） | 必須 |
| **住所** | 申告住所（確認可能な範囲） | 推奨 |
| **SNSアカウント** | 申告URL・検索で判明したアカウント | 推奨 |

**補足**

- 屋号のみで活動する場合は、屋号と氏名の **両方** で検索する
- 住所は公開情報・HP・登記・申告内容の整合確認に利用する（過度な個人情報収集は行わない）

# 4. 基本確認項目

審査担当者は、区分に応じて該当する項目を **YES / NO** で記録する。
**必須項目が NO の場合は、原則「要確認」とし、追加確認を実施する。**

| # | チェック項目 | 法人 | 個人 | YES の基準 |
|---|-------------|------|------|-----------|
| 1 | □ 法人番号を確認した | ○ | — | 法人番号公表サイト等で番号・商号が申告と一致 |
| 2 | □ 会社実在を確認した | ○ | ○ | 登記・HP・電話・住所等で事業体の実在を確認 |
| 3 | □ HPを確認した | 推奨 | 推奨 | 公式サイトまたは同等の事業紹介ページを確認（ない場合は備考に理由） |
| 4 | □ 電話番号を確認した | ○ | ○ | 申告番号に架電し、事業体・担当者と確認できた |
| 5 | □ メールアドレスを確認した | ○ | ○ | 申告アドレスへの連絡が届き、申請者と確認できた |
| 6 | □ 所在地を確認した | ○ | 推奨 | 申告住所と公開情報・登記等が矛盾しない |
| 7 | □ SNSを確認した | 推奨 | 推奨 | 第6章に基づき確認実施（アカウントなしは備考に記載） |
| 8 | □ 過去実績を確認した | 推奨 | 推奨 | 申告実績・紹介先・施工事例等で事業継続性を確認 |

**判定の目安**

- 必須相当項目（1・2・4・5）がすべて YES → 基本確認は通過
- いずれかが NO → **要確認**（情報不足）。追加確認後に再判定

# 5. 公開情報調査

## 5.1 調査方針

- 業務上必要な範囲で、**公開されている情報** を調査する
- 検索結果の URL・日付・概要を証跡として記録する（第11章・第12章）
- 風評・口コミのみを根拠に「該当」と判定しない。公式情報・報道・行政公表等を重視する

## 5.2 Google検索例

### 会社名・屋号で検索

```
「{会社名}」
「{会社名}」 評判
「{会社名}」 行政処分
「{会社名}」 逮捕
「{会社名}」 反社
「{会社名}」 暴力団
「{会社名}」 詐欺
「{会社名}」 トラブル
「{会社名}」 消費者庁
```

### 代表者名・申請者氏名で検索

```
「{代表者名}」
「{代表者名}」 逮捕
「{代表者名}」 詐欺
「{代表者名}」 行政処分
「{代表者名}」 暴力団
「{代表者名}」 {会社名}
```

### 屋号（個人）で検索

```
「{屋号}」
「{屋号}」 評判
「{屋号}」 トラブル
「{氏名}」 「{屋号}」
```

## 5.3 確認結果記録欄（テンプレート）

審査担当者は、調査実施ごとに以下を記録する。

| 項目 | 記入内容 |
|------|----------|
| 検索実施日 | YYYY-MM-DD |
| 検索キーワード | 上記例から使用した語句 |
| 確認したURL（最大5件） | https://... |
| 概要メモ | 該当なし / 要確認の理由 / 該当の根拠（50〜200字程度） |
| 判定への影響 | なし / 要確認理由 / 該当根拠 |

**記録例**

```
[公開情報調査] 2026-06-23 担当: 佐藤
キーワード: 「株式会社サンプル建設」逮捕 / 反社 / 行政処分
URL: （該当なし）
概要: 公式・報道・行政公表において反社関連情報は確認されず。
判定への影響: なし
```

# 6. SNS確認

## 6.1 確認対象プラットフォーム（例）

申請フォームに記載の SNS URL を優先し、ない場合は検索で判明したアカウントを確認する。

| プラットフォーム | 確認優先度 |
|-----------------|-----------|
| Facebook | 中 |
| Instagram | 中 |
| X（旧Twitter） | 高 |
| TikTok | 中 |
| YouTube | 低〜中 |
| LinkedIn | 低〜中 |

## 6.2 確認ポイント

以下のいずれかに該当する投稿・プロフィール表現がある場合は、**要確認** または **該当** の判断材料とする。

| # | 確認ポイント | 判定の目安 |
|---|-------------|-----------|
| 1 | **違法行業・違法行為の示唆** | 明確な示唆 → 要確認以上。継続的・組織的 → 該当検討 |
| 2 | **暴力団関係の示唆** | 明示・象徴等 → 該当検討 |
| 3 | **詐欺的表現** | 誇大・虚偽の繰り返し → 要確認。組織的詐欺 → 該当検討 |
| 4 | **著しいトラブル投稿** | 事実不明 → 要確認。報道等と整合 → 該当検討 |
| 5 | **犯罪自慢・違法自慢** | 合理的に信じられる内容 → 該当検討 |
| 6 | **脅迫的表現** | 取引相手・第三者への脅迫 → 該当検討 |

## 6.3 SNS確認の記録

| 項目 | 記入 |
|------|------|
| 確認したアカウントURL | |
| 公開 / 非公開 | 非公開の場合は要確認（H09） |
| 確認日 | |
| 問題の有無 | なし / 要確認 / 該当 |
| スクリーンショット保存 | 要確認・該当時は保存推奨（個人情報に配慮） |

# 7. 行政処分確認

## 7.1 確認対象

| 区分 | 確認対象 | 該当業種 |
|------|----------|----------|
| 建設業 | 建設業許可行政処分（指示・停止・取消等） | 建設関連業種 |
| 指名停止 | 国・地方公共団体の入札参加停止・指名停止 | 公共工事関与時 |
| 業務停止命令 | 監督官庁による業務停止 | 許認可業種全般 |
| 消費者庁 | 措置命令・公表事案 | 全業種 |
| その他官公庁 | 国土交通省・厚生労働省・警察等の公表情報 | 業種に応じて |

## 7.2 確認方法

| 方法 | 参照先・手順 |
|------|-------------|
| 建設業許可処分 | 各省庁・都道府県の建設業者情報検索、建設工事紛争調整委員会等 |
| 消費者庁公表 | [消費者庁ウェブサイト](https://www.caa.go.jp/) の措置命令・公表事例 |
| 入札参加停止 | 各地方自治体の公表情報 |
| 一般検索 | 第5章の検索キーワードに「行政処分」「業務停止」「許可取消」を追加 |

## 7.3 行政処分の判定

| 状況 | 判定 |
|------|------|
| 反社・暴力団関連の行政処分・公表 | **該当** または **否認（R01/R08）** |
| 重大な許可取消・継続的業務停止 | **要確認** → 審査基準 R08 と合わせて判断 |
| 軽微な指導・過去の単発処分で改善済み | **問題なし** または **要確認**（個別判断、備考必須） |
| 確認できず | **要確認**（H09） |

# 8. 反社自己申告

登録フォームの確認・同意事項および将来の電子契約書と整合する自己申告内容を以下に定める。

## 8.1 登録フォームとの対応

登録フォーム（`partner-register-form.js`）では、申請者が以下に同意している。

| フォーム項目 | 申告内容 |
|-------------|----------|
| `agree_antisocial` | **反社会的勢力ではありません** |
| `agree_truth` | **提出情報に虚偽はありません** |

本チェックシートおよび電子契約では、上記を以下の5項目に展開して確認する。

## 8.2 自己申告確認事項（YES / NO）

申請記録上、申請者が以下を表明しているものとして審査する。

| # | 確認事項 | 審査担当の確認 |
|---|----------|---------------|
| 1 | □ **反社会的勢力ではない** | 公開情報調査・SNS等と矛盾がないか |
| 2 | □ **反社会的勢力の関係者ではない** | 代表者・役員・実質支配者について確認 |
| 3 | □ **反社会的勢力に資金提供を行っていない** | 明らかな矛盾がないか（通常は申告ベース） |
| 4 | □ **反社会的勢力を利用していない** | 明らかな矛盾がないか |
| 5 | □ **将来も反社会的勢力と関与しない** | 契約条項と整合（電子契約時に再確認推奨） |

## 8.3 自己申告と調査結果が矛盾した場合

| 状況 | 対応 |
|------|------|
| 軽微な矛盾・説明可能 | **要確認**（申請者に説明を求める） |
| 明らかな虚偽 | **該当** → 否認 **R01** および / または **R10**（同意事項違反） |

## 8.4 電子契約での再確認（将来）

電子契約締結時に、契約書へ以下の表明条項を盛り込み、申請時申告との整合を取る。

- 反社会的勢力排除に関する表明・確約
- 違反時の契約解除条項

# 9. 要確認基準

## 9.1 要確認にする条件

以下のいずれかに該当する場合、反社判定を **「要確認」** とし、審査ステータスを **保留（H09）** とする。

| コード | 条件 | 例 |
|--------|------|-----|
| A01 | **情報不足** | HPなし・電話未確認・実在未確認 |
| A02 | **検索結果が曖昧** | 類似名称の別法人・別人事業が多数ヒット |
| A03 | **同姓同名** | 代表者名で逮捕記事等がヒットするが同一性不明 |
| A04 | **ネガティブ情報あるが事実不明** | 掲示板・SNSの批判のみで裏取りなし |
| A05 | **SNS非公開・削除** | 確認ができない |
| A06 | **外部API要確認フラグ** | 将来導入するスクリーニングAPIが review を返した |
| A07 | **役員・実質支配者未確認** | 法人で追加調査が必要 |
| A08 | **行政処分の詳細不明** | 処分の内容・期間・業種該当性の確認が必要 |

## 9.2 追加確認フロー

```
要確認（review）判定
    ↓
① 備考に要確認コード（A01〜A08）を記録
    ↓
② ステータスを「保留」（H09）に設定
    ↓
③ 追加確認の実施
    │  ・申請者への質問・追加資料依頼
    │  ・上長へのエスカレーション
    │  ・外部API再照会（将来）
    │  ・報道・公表情報の詳細確認
    ↓
④ 再判定
    ├→ 問題なし（clear）→ 審査継続
    ├→ 要確認継続 → 保留維持（SLA: 30営業日）
    └→ 該当（hit）→ 否認（R01）
```

## 9.3 エスカレーション基準

以下の場合は、審査担当者から **運営管理者・法務担当** へエスカレーションする。

- 報道・公表情報で反社関連の疑いが強い
- 同姓同名の判断が困難で否認・承認の影響が大きい
- 外部APIと手動調査の結果が矛盾する
- 申請者から反論・説明資料が提出された

# 10. 該当基準

## 10.1 即否認対象（該当 / hit）

以下のいずれかに該当し、**合理的に認められる** 場合、反社判定を **「該当」** とし、**即否認（R01）** とする。

| コード | 該当条件 |
|--------|----------|
| B01 | **反社認定情報** — 反社DB・公的資料で該当が確認された |
| B02 | **暴力団関係情報** — 暴力団員・準構成員・関係企業との関与が確認された |
| B03 | **犯罪組織関与** — 組織的犯罪への関与が確認された |
| B04 | **マネーロンダリング関与** — 公的調査・報道等で合理的に認められる |
| B05 | **継続的な重大違法行為** — 詐欺・恐喝・脅迫等の組織的・継続的行為が確認された |
| B06 | **自己申告の明らかな虚偽** — 反社でないと申告したが関与が確認された（R10併記可） |

## 10.2 審査基準との連携（Rコード）

| 本チェックシート | 審査基準 | 内容 |
|-----------------|----------|------|
| 該当（hit） | **R01** 反社該当 | 再申請不可 |
| 該当（hit）+ 虚偽申告 | **R10** 同意事項違反 | 再申請不可 |
| 要確認（review） | **H09** 反社チェック追加確認中 | 保留 |
| 問題なし（clear） | 審査基準 第9章 項目1 YES | 反社チェック完了 |

## 10.3 該当判定時の必須記録

否認（R01）とする場合、以下を Builder 備考に必ず記録する。

- 該当コード（B01〜B06）
- 根拠となった情報源（URL・公表日・資料名）
- 判定者・判定日
- 上長承認の有無（B01〜B05 は原則要記録）

# 11. Builder管理連携

## 11.1 現在管理する項目

Builder 管理画面（`/builder/partner-management.html`）および将来の本番DBで、反社チェック関連を以下の項目で管理する。
審査基準 第8章の「反社」フィールドと整合させる。

| 項目名 | 型・形式 | 必須 | 説明 |
|--------|----------|------|------|
| **反社チェック日** | date (YYYY-MM-DD) | ○ | 最終確認を完了した日 |
| **担当者** | string | ○ | チェック実施者氏名またはID |
| **判定** | enum: `clear` / `review` / `hit` | ○ | 問題なし / 要確認 / 該当 |
| **確認方法** | string[] | ○ | 例: `manual_search`, `phone`, `sns`, `registry`, `api` |
| **証跡URL** | text | 推奨 | 調査で参照したURL（複数可、改行区切り） |
| **備考** | text | ○ | 要確認コード・該当コード・調査概要 |

### 審査基準「反社」フィールドとの対応

| 本チェックシート判定 | Builder「反社」enum |
|---------------------|---------------------|
| 未実施 | 未確認 |
| 問題なし（clear） | 問題なし |
| 要確認（review） | 要確認 |
| 該当（hit） | 該当 |

## 11.2 将来追加予定項目

外部反社チェックサービス / API 導入時に追加する項目。

| 項目名 | 説明 |
|--------|------|
| **API照会ID** | 外部サービスの照会リクエストID |
| **API照会日時** | ISO 8601形式 |
| **API照会結果** | clear / review / hit の raw レスポンス |
| **スクリーニング結果** | ヒット件数・マッチスコア・該当リスト |
| **API提供者** | サービス名（例: 日経テレコン、RIVS等） |
| **再照会日** | 定期再スクリーニング用 |

確認方法: manual_search, phone, registry, sns

要確認コード: —
該当コード: —
メモ: 法人番号・代表者名で検索。反社・暴力団関連情報なし。電話で代表者確認済み。

# 13. 保管方針

## 13.1 保存期間（推奨）

| データ種別 | 保存期間 |
|-----------|----------|
| 反社チェック記録票（写し・電子データ） | **契約期間中** + **契約終了後5年間** |
| 証跡URL・調査メモ・スクリーンショット | 上記と同様 |
| 外部API照会結果 | 上記と同様（将来） |
| 否認（R01）の記録 | **契約終了後5年間**（再申請防止・説明責任のため） |

## 13.2 保管場所・方法

| 項目 | 方針 |
|------|------|
| 一次保管 | Builder 管理画面の備考・専用フィールド（将来） |
| 二次保管 | 運営用ストレージ（アクセス権限を限定） |
| 証跡ファイル | 個人情報を含むため、暗号化・権限管理を推奨 |

## 13.3 個人情報保護との整合

- **収集の最小化** — 審査に必要な範囲の情報のみ収集・記録する
- **利用目的の限定** — 協力パートナー審査・契約管理・法令遵守の目的に限定する
- **第三者提供の制限** — 外部反社API利用時は委託先との契約・プライバシーポリシー整備を行う
- **開示・削除** — 個人情報保護法および `IWASHOプライバシーポリシー` / `TASFULプライバシーポリシー` に基づき対応する
- **記録の適正化** — 風評・憶測のみを根拠にした記録は避け、事実に基づく記載とする

## 13.4 契約終了後の取扱い

- 契約終了後5年経過した記録は、法令で longer の保存が求められない限り、削除または匿名化する
- 否認・該当（R01）記録は、再申請防止のため期間内は慎重に管理する
