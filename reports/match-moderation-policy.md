## 1. 目的

- ユーザー間トラブル・違反行為に対し、**一貫した判断**で対応する
- 通報審査・プロフィール停止・allowlist 停止の基準を明文化する
- 自動モデレーション未実装の β フェーズにおける **人的判断**の補助とする

---

## 2. 対応レベル定義

| レベル | 意味 | 典型操作 |
|--------|------|----------|
| **注意** | 軽微・初回・疑い程度 | 運用メモ · 通報 `dismissed` または様子見 |
| **警告** | 再発防止が必要 | ユーザーへの警告連絡（メール等）· 通報 `resolved` |
| **一時停止** | 利用停止（期間限定または審査完了まで） | `profile_status=suspended` · 必要なら allowlist `revoked` |
| **永久停止** | β/サービスからの排除 | allowlist `revoked` + `suspended` + 再招待なし方針 |
| **通報却下** | 根拠不十分・誤通報・軽微 | 通報 `dismissed` |
| **確認保留** | 証拠不足・追加調査必要 | 通報 open 維持 or ユーザー停止せず調査 · 本人確認 pending 維持 |

**技術的制約（β）:**

- 通報提出だけでは **自動ブロック・自動停止しない**
- `match_has_active_match_ban()` は現状 **常に false**（ban テーブル未稼働）
- 制裁は **管理画面の suspend / allowlist revoked** が主手段

---

## 3. 判断基準（横断）

| 基準 | 意味 | 典型対応レベル |
|------|------|----------------|
| **即時停止** | 被害拡大・法令違反の疑いが強い | 一時停止〜永久停止 |
| **要確認** | 証拠・本人確認・相手証言が必要 | 確認保留 → 審査後に判断 |
| **軽微** | 初回・悪意なし・誤解の範囲 | 注意〜警告 |
| **却下可能** | 通報根拠なし・利用規約違反に当たらない | 通報却下 |

## 4. 違反分類と対応マトリクス

凡例: **判断** = 即時停止 / 要確認 / 軽微 / 却下可能  
**対応** = 推奨対応レベル（複数可）

### 4.1 18歳未満利用

| 項目 | 内容 |
|------|------|
| 例 | プロフィール・申告から18歳未満と判明、年齢確認で矛盾 |
| 判断 | **即時停止** |
| 対応 | **永久停止** · allowlist `revoked` · 年齢確認却下 · 法務エスカレーション |
| UI 通報理由 | `other` または該当なし → 運用で分類 |
| 備考 | 児童保護関連は最優先 |

### 4.2 なりすまし

| 項目 | 内容 |
|------|------|
| 例 | 他人の写真・名前・業者になりすまし |
| 判断 | **即時停止**（明確な場合） / **要確認**（疑い） |
| 対応 | 一時停止〜永久停止 · 本人確認却下 · 通報 `resolved` |
| UI 通報理由 | `impersonation` |

### 4.3 虚偽プロフィール

| 項目 | 内容 |
|------|------|
| 例 | 年齢・職業・目的の著しい虚偽（悪意ある場合） |
| 判断 | **要確認** 〜 **即時停止**（悪質時） |
| 対応 | 警告 · 一時停止 · プロフィール修正依頼 |
| UI 通報理由 | `other` |

### 4.4 外部誘導

| 項目 | 内容 |
|------|------|
| 例 | LINE・Instagram・外部マッチングアプリ・個人サイトへの誘導（営利/収集目的） |
| 判断 | **要確認** 〜 **即時停止**（悪質・大量時） |
| 対応 | 警告 · 一時停止 |
| UI 通報理由 | `other` / `inappropriate_message` |

### 4.5 金銭要求

| 項目 | 内容 |
|------|------|
| 例 | 初対面で金銭・振込・投資・借金の要求 |
| 判断 | **即時停止** |
| 対応 | 一時停止〜永久停止 · 通報 `resolved` · 被害者には警察相談案内を検討 |
| UI 通報理由 | `other` / `harassment` |

### 4.6 営業 / 勧誘

| 項目 | 内容 |
|------|------|
| 例 | 保険・投資・副業・宗教・建設営業の一方的勧誘 |
| 判断 | **軽微**（初回）〜 **要確認**（継続） |
| 対応 | 警告 · 一時停止（反復時） |
| UI 通報理由 | `other` / `inappropriate_message` |
| 備考 | 建設業界向けサービスのため、**正当なビジネス目的**と **迷惑勧誘**の線引き要確認 |

### 4.7 性的・卑猥表現

| 項目 | 内容 |
|------|------|
| 例 | プロフィール・TALK での露骨な性的要求・画像 |
| 判断 | **即時停止**（露骨・強制） / **要確認**（文脈依存） |
| 対応 | 一時停止〜永久停止 · 通報 `resolved` |
| UI 通報理由 | `inappropriate_message` / `harassment` |
| 備考 | TALK 内コンテンツは TALK モデレーションも参照 |

### 4.8 誹謗中傷

| 項目 | 内容 |
|------|------|
| 例 | 人格否定・差別的発言・公開侮辱 |
| 判断 | **要確認** 〜 **即時停止**（悪質・反復） |
| 対応 | 警告 · 一時停止 |
| UI 通報理由 | `harassment` / `inappropriate_message` |

### 4.9 脅迫

| 項目 | 内容 |
|------|------|
| 例 | 危害の示唆・晒し脅し・金銭をちらつかせる威圧 |
| 判断 | **即時停止** |
| 対応 | 永久停止 · 法務・当局相談検討 |
| UI 通報理由 | `harassment` |

### 4.10 ストーカー行為

| 項目 | 内容 |
|------|------|
| 例 | ブロック後の執拗な接触・複数アカウント・足跡の悪用 |
| 判断 | **即時停止** |
| 対応 | 永久停止 · allowlist `revoked` · 被害者にブロック案内 |
| UI 通報理由 | `harassment` |

### 4.11 個人情報要求

| 項目 | 内容 |
|------|------|
| 例 | 初対面で住所・勤務先・マイナンバー・口座の要求 |
| 判断 | **要確認** 〜 **即時停止**（詐欺連動時） |
| 対応 | 警告 · 一時停止 |
| UI 通報理由 | `other` |

### 4.12 迷惑行為

| 項目 | 内容 |
|------|------|
| 例 | スパム like · 大量メッセージ · 明らかないたずら |
| 判断 | **軽微** 〜 **要確認** |
| 対応 | 注意 · 警告 · 一時停止（反復） |
| UI 通報理由 | `other` / `inappropriate_message` |

### 4.13 詐欺疑い

| 項目 | 内容 |
|------|------|
| 例 | なりすまし + 金銭 · 架空工事 · フィッシング URL |
| 判断 | **即時停止** |
| 対応 | 永久停止 · 通報 `resolved` · エスカレーション |
| UI 通報理由 | `other` / `impersonation` |

### 4.14 反社 / 違法行為

| 項目 | 内容 |
|------|------|
| 例 | 違法労働・薬物・暴力団関与の示唆 |
| 判断 | **即時停止** |
| 対応 | 永久停止 · 法務エスカレーション |
| UI 通報理由 | `other` |

## 5. 通報理由（UI）と違反分類の対応

| UI 通報理由（コード） | 主に紐づく違反分類 |
|----------------------|-------------------|
| `inappropriate_message` | 性的表現 · 誹謗中傷 · 迷惑行為 · 外部誘導 |
| `impersonation` | なりすまし · 詐欺疑い |
| `harassment` | ハラスメント全般 · 脅迫 · ストーカー · 金銭要求 |
| `other` | 上記以外 · 営業勧誘 · 個人情報 · 18歳未満等 |

運用者は **詳細テキスト（detail）** を読み、上記 14 分類にマッピングする。

---

## 6. エスカレーション基準

以下は運用リード + 法務（または外部専門家）へエスカレーション:

- 18歳未満・児童関連
- 刑事告訴の可能性
- 個人情報漏洩疑い
- メディア・行政からの照会
- 同一ユーザーへの複数通報（3件以上 / 7日以内を目安）
- 管理者権限の疑い・内部不正

---

## 7. 対応フロー（簡易）

```
通報 open 受信
    ↓
重大度判定（即時停止？）
    ↓ はい → suspend + revoked + resolved 記録
    ↓ いいえ
証拠確認（プロフィール・TALK・本人確認）
    ↓
対応レベル決定（注意/警告/停止/却下/保留）
    ↓
管理画面で実行 + 監査ログ確認
    ↓
必要ならユーザー通知
```

---

## 8. β フェーズの制限事項

| 制限 | 影響 |
|------|------|
| AI 自動監視なし | 全件人的レビュー依存 |
| 書類画像未保存 | 本人確認は別チャネル補完が必要な場合あり |
| ブロック解除 API なし | 誤ブロックは技術対応 |
| ban テーブル未稼働 | 永久停止は allowlist + suspend で代替 |
| 通報→自動制裁なし | 対応漏れリスク — 日次キュー確認必須 |

## 9. 改訂

本ポリシーは β 運用のフィードバックに応じて改訂する。改訂時は版番号と変更履歴を追記すること。
