## 0. エグゼクティブサマリー

TASFUL エコシステムに **素材ダウンロード型の新サービス**（仮称: **TASFUL Materials**）を追加する。  
MVP は **検索ファースト · 広告付き無料 DL** に集中し、Platform（取引）· Builder（案件）· TLV（Live）· AI 各 surface とは **別 UI · 別データ境界** とする（AD-001 思想に準拠）。

| 観点 | 推奨 |
| --- | --- |
| **おすすめ構成** | Cloudflare Pages + Supabase で MVP → 成長後 R2 + Worker 署名 URL |
| **MVP カテゴリ** | アイコン · 壁紙 · AIプロンプト（AI 大量生成向き · 法務リスク低） |
| **第2波** | BGM · 効果音 · SNS/サムネテンプレ |
| **第3波** | イラスト（人キュレーション中心）· Premium · クリエイター販売 |
| **将来性** | 高（SEO · リピート DL · Premium/サブスク · 横断 AI 生成パイプライン） |
| **主な懸念** | ライセンス表示 · 著作権 · AI 生成物の商用可否 · ストレージコスト · 重複/品質 |

### 1.1 レイヤー図

```

│ 基本DL       │ 高品質/商用可  │ 手数料モデル  │ 優先DL/限定素材  │
└──────────────┴──────────────┴──────────────┴────────────────┘
         │              │              │
         └──────────────┴──────────────┘
```

### 1.2 各モード定義

| モード | ユーザー価値 | 収益 | MVP |
| --- | --- | --- | --- |
| **無料（広告あり）** | 登録/非登録で基本 DL · プレビュー · 個人利用中心 | ディスプレイ · ダウンロード前後インタースティシャル · ネイティブ枠 | ✅ コア |
| **Premium（広告なし）** | 広告非表示 · 一部高解像度 · 商用可素材パック · 一括 DL | 月額/年額 · 単品アップセル | 📋 Phase 2 |
| **クリエイター販売** | 作者が価格設定 · TASFUL が決済/配信代行 | 販売手数料 10〜20%（要法務） | 📋 Phase 3 |
| **サブスク（将来）** | 全カタログ · 月次クレジット · 新着先行 | MRR · 解約率管理 | 📋 Phase 4 |

### 1.3 既存 TASFUL との関係

| 既存サービス | 関係 |
| --- | --- |
| **Platform** | 別目的（取引/掲載）。素材は **導線のみ**（「この素材で出品画像を作る」等）· 統合 DB 混在しない |
| **Builder** | 現場写真/案件とは別。将来「Builder AI 用テンプレ素材」リンク程度 |
| **TASFUL AI** | 素材生成パイプラインの **バックエンド候補**（Gateway 経由 · 別 surface） |
| **TLV** | BGM/効果音の **利用先** として参照可。TLV 専用 AI は作らない（AD-004） |
| **AI 秘書** | 運営向け: 通報/著作権申立/品質キュー監視（将来） |

**SNS 機能は作らない** — フォロー/コレクションは **素材発見** 用途に限定（タイムライン · DM · いいね数競争なし）。

## 4. AI 大量生成運用

### 4.1 カテゴリ分類

| 区分 | カテゴリ | 運用 |
| --- | --- | --- |
| **AI 大量生成向き** | 壁紙 · アイコン · AIプロンプト · BGM（生成音楽）· サムネ/SNS テンプレ骨格 | バッチ生成 → 自動メタ付与 → 品質スコア → 公開キュー |
| **人投稿中心** | 効果音（実録）· イラスト（ブランド系）· 商用ライセンス系 | 作者アップロード → モデレーション → 公開 |
| **ハイブリッド** | BGM · イラスト · テンプレ | AI 下書き + 人が修正/選別 |

### 4.2 AI 生成パイプライン（Skeleton）

```
[生成ジョブ定義]
      ↓
[Gateway / 専用 Edge — TASFUL AI とは surface 分離]
      ↓
[アセット保存 — Storage/R2]
      ↓
[メタ生成 — タイトル/タグ/カテゴリ/ライセンス]
      ↓
[重複チェック]
      ↓
[品質チェック — 自動 + サンプリング人レビュー]
      ↓
[公開フロー — draft → review → published]
```

### 4.3 重複チェック

| 手法 | 用途 |
| --- | --- |
| ファイルハッシュ（SHA-256） | 完全一致 |
| 知覚ハッシュ（pHash/dHash） | 画像/壁紙/イラスト近似 |
| 音声 fingerprint | BGM/SFX 近似 |
| テキスト正規化 + simhash | AIプロンプト |
| タイトル+タグ Jaccard | 弱い重複 |

**方針:** 完全一致は自動ブロック · 近似は `review` キュー。

### 4.4 品質チェック

| 自動 | 人レビュー |
| --- | --- |
| 解像度/形式/ファイルサイズ | 商用カテゴリ |
| 音声長/無音/クリッピング | 通報素材 |
| AI 生成アーティファクト検出（将来） | ランダム 5% サンプル |
| NSFW スコア（将来 Edge） | イラスト/壁紙 |

### 4.5 公開フロー

| 状態 | 説明 |
| --- | --- |
| `draft` | 生成直後 · 非公開 |
| `review` | 重複/品質フラグ |
| `published` | 検索可能 |
| `archived` | DL 停止 · 履歴保持 |
| `rejected` | 再処理不可 |

## 6. インフラ

### 6.1 MVP 開始構成（推奨: 可能）

| 層 | 技術 | 役割 |
| --- | --- | --- |
| **UI** | Cloudflare Pages | 静的 HTML/JS · 既存 `build:pages` 流用 |
| **API** | Supabase Edge Functions または CF Pages Functions | 検索 API · DL カウント · 署名 URL（将来） |
| **DB** | Supabase Postgres | 素材メタ · タグ · ユーザー · 履歴 |
| **Auth** | Supabase Auth | お気に入り/履歴/Premium |
| **Storage** | Supabase Storage | 画像/音声/小〜中 ZIP（既存 `listing-images` パターン流用） |
| **CDN** | Supabase public URL + CF キャッシュ | MVP 十分 |

**開始可能理由:** 既に Supabase Storage + Postgres + CF Pages の運用実績あり。新バケット `materials-assets` + RLS で開始できる。

### 6.2 Cloudflare R2 移行タイミング

| トリガー | 目安 |
| --- | --- |
| 総アセット容量 | **> 50 GB** または Supabase Storage プラン上限接近 |
| 月間 DL 数 | **> 100 万** または egress 課金が広告収益の 20% 超 |
| カテゴリ追加 | **4K 動画 · 大容量 ZIP · 3D** を本格提供 |
| 署名 URL 必須 | Premium/有料 · ホットリンク防止 |
| グローバル latency | 音声/動画の TTFB 要件 |

**移行方針:** R2 を primary · Supabase はメタ/Auth 正本 · Worker で署名 URL 発行 · 既存 URL は 301/互換レイヤ

### 6.3 検索索引

| MVP | 将来 |
| --- | --- |
| Postgres `tsvector` + タグ GIN | Meilisearch / Typesense（自ホスト or Cloud） |
| カテゴリ/タグフィルター SQL | ベクトル検索（類似素材） |

### 8.2 AD-012 準拠

- 小学生でも分かるカテゴリ名（「効果音」 not 「SFX Assets」）
- 迷ったら **検索1本** · カテゴリは補助
- AI 生成はバッジで明示 · 商用可否は **色+短文**（緑=個人可 等）
- 新機能でホームを複雑化しない — Premium/販売は別タブ

### 8.3 作らない UI

- タイムライン · フォローフィード · DM · いいねランキング · UGC コメントスレッド

## 12. 推奨まとめ

### おすすめ構成

1. **仮称 TASFUL Materials** — `materials/` 独立 surface  
2. **MVP:** 検索 + 3 カテゴリ（アイコン/壁紙/AIプロンプト）+ Supabase Storage  
3. **収益:** 広告のみ → Premium → クリエイター  
4. **AI:** バッチ生成パイプラインは Phase 3 · 最初は手動/CSV 投入可  

### 優先順位

1. P0 検索/カテゴリ/DL/ライセンス  
2. P1 フィルター/人気/新着/広告  
3. P2 おすすめ/AI バッジ/運営 CMS  
4. P3 Premium/販売/サブスク  

### 将来性

- SEO 流入 · 素材需要は Builder/TLV/AI ユーザーと **相性良**  
- Premium/商用ライセンス · クリエイター economy  
- TASFUL AI 生成物の **配布出口** としてエコシステム強化  

### 懸念点（再掲）

ライセンス · ストレージコスト · AI 品質 · スコープ分離 · 法務（商用/AI学習素材）
