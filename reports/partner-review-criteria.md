# 協力パートナー審査基準（IWASHO / TASFUL 共通）

| 項目 | 内容 |
|------|------|
| 文書バージョン | 1.0 |
| 制定日 | 2026-06-23 |
| 適用範囲 | IWASHO・TASFUL・Builder 経由の協力パートナー登録申請すべて |
| 対象区分 | 法人 / 個人事業主 / 一人親方 / フリーランス |
| 関連画面 | `/iwasho/partner-register.html` `/partner-register.html` `/builder/partner-management.html` |

---

## 文書の目的

本基準は、協力パートナー登録申請に対する審査判断のばらつきをなくし、審査担当者・運営・Builder 管理画面が同一ルールで運用できるように定めるものである。

- 審査ステータスの定義と遷移条件を明確化する
- 承認・保留・否認の判断基準を共通化する
- 電子契約締結および案件紹介開始の前提条件を定める
- 将来の反社チェック API・電子契約サービス・案件管理との接続を想定したデータ項目を定義する

---

# 1. 審査ステータス定義

Builder 管理画面および運営審査で使用するステータスは以下 **5段階** とする。内部コード（Builder 連携用）は括弧内に記載。

### 1.1 ステータス一覧

| ステータス | 内部コード | 意味 |
|-----------|-----------|------|
| **審査待ち** | `pending` | 登録申請を受付済み。一次確認・各種チェックが未完了の状態 |
| **保留** | `hold` | 審査継続中だが、追加情報・書類・確認待ちにより判断を保留している状態 |
| **承認** | `approved` | 承認必須条件をすべて満たし、電子契約手続きへ進める状態 |
| **否認** | `rejected` | 必須条件未達・反社懸念・虚偽申告等により登録を認めない状態 |
| **契約済み** | `contracted` | 電子契約締結が完了し、案件紹介・マッチング対象として有効な状態 |

### 1.2 ステータス遷移図

```
登録申請受付
    ↓
審査待ち (pending)
    ├→ 保留 (hold) ──────────────┐
    │       ↓（保留解除）          │
    ├→ 承認 (approved)            │
    │       ↓                     │
    │   契約済み (contracted)     │
    │                             │
    └→ 否認 (rejected) ←──────────┘（再申請不可条件に該当した場合）
```

### 1.3 各ステータスの遷移条件

#### 審査待ち（pending）

| 項目 | 内容 |
|------|------|
| **設定タイミング** | 登録フォーム送信直後（自動設定） |
| **状態の意味** | 申請データは受付済み。審査担当による確認作業が開始されていない、または作業途中の初期状態 |
| **次に遷移できるステータス** | 保留 / 承認 / 否認 |
| **担当者アクション** | 一次確認の着手、反社チェック・書類確認の実施 |

#### 保留（hold）

| 項目 | 内容 |
|------|------|
| **設定タイミング** | 審査途中で判断に必要な情報・書類が不足している場合 |
| **代表的な理由** | 添付書類不足、保険証券未提出、インボイス番号の国税庁照合待ち、実績確認中、電話確認未実施、資格証の真正性確認待ち |
| **状態の意味** | 否認ではないが、現時点では承認判断ができない。申請者への追加依頼・内部確認が必要 |
| **次に遷移できるステータス** | 審査待ち（再確認中）/ 承認 / 否認 |
| **SLA目安** | 保留理由発生から **14営業日以内** に追加提出を依頼。 **30営業日** 経過しても未対応の場合は否認または申請者へ最終確認のうえクローズ |

#### 承認（approved）

| 項目 | 内容 |
|------|------|
| **設定タイミング** | 第2章「承認必須条件」をすべて満たし、第9章チェックリストがすべて YES の場合 |
| **状態の意味** | 協力パートナーとしての審査は合格。電子契約の案内・締結手続きへ進める |
| **次に遷移できるステータス** | 契約済み / 否認（契約前に重大な事実判明時のみ。要管理者承認） |
| **注意** | 承認は案件紹介開始を意味しない。案件紹介は **契約済み** 以降 |

#### 否認（rejected）

| 項目 | 内容 |
|------|------|
| **設定タイミング** | 第4章「否認条件」に該当した場合 |
| **代表的な理由** | 反社該当、虚偽申告、インボイス未登録、保険未加入、労災未加入、連絡不能、重大な行政処分歴、重大な過去トラブル歴 |
| **状態の意味** | 当該申請は登録不可。理由を申請者へ通知（テンプレート使用） |
| **次に遷移できるステータス** | 原則なし（再申請は第4章の再申請条件に従う） |

#### 契約済み（contracted）

| 項目 | 内容 |
|------|------|
| **設定タイミング** | 電子契約サービス上で協力パートナー契約の締結が完了した時点（手動または API 連携で更新） |
| **状態の意味** | 正式な協力パートナー。案件紹介・マッチング・Builder 上のパートナー検索対象として有効 |
| **次に遷移できるステータス** | 契約解除・停止は別途「契約状況」フィールドで管理（ステータス自体は維持し備考に記録） |

### 1.4 ステータス変更時の記録義務

ステータスを変更する際、Builder 管理画面の **備考** に以下を必ず記録する。

- 変更日時
- 変更者（審査担当者名）
- 変更前 → 変更後ステータス
- 変更理由（保留理由コード / 否認理由コード / 承認確認番号 等）

# 2. 承認必須条件

以下の条件を **すべて満たした場合のみ** ステータスを「承認」にできる。

### 2.1 共通必須条件

| # | 条件 | 確認方法 | 判定基準 |
|---|------|----------|----------|
| 1 | **反社チェック問題なし** | 反社DB照合（将来API）+ 担当者確認 | 反社・暴力団関係者・反社該当企業との取引懸念がないこと |
| 2 | **インボイス登録済み** | 国税庁公表サイト照合 + 申請番号一致 | 適格請求書発行事業者登録番号（T+13桁）が有効であること |
| 3 | **保険加入済み** | 保険証券の提出・内容確認 | 請負業者賠償責任保険（または同等の事業賠償責任保険）に加入していること |
| 4 | **労災加入済み** | 労災加入証明・申告内容確認 | 区分に応じた労災加入要件を満たすこと（2.3参照） |
| 5 | **連絡先確認済み** | 電話またはメールによる本人確認 | 申請者・担当者へ連絡が取れ、申請内容の意思確認が完了していること |
| 6 | **実在確認済み** | 登記・開業届・Webサイト・口座名義等 | 申請した事業体が実在し、申告住所・屋号と整合すること |

### 2.2 保険条件（数値基準）

| 項目 | 最低基準 |
|------|----------|
| 対人補償額 | **1億円以上** |
| 対物補償額 | **5,000万円以上** |
| 保険期間 | 審査時点で **有効期限内** であること |
| 被保険者 | 申請事業体（法人名・屋号）と一致、または被保険者範囲に申請者が含まれること |

**例外（保留扱い・承認不可）**

- 対人・対物のいずれかが基準未満 → **否認**（再申請は保険更新後）
- 保険証券未提出 → **保留**
- 「加入予定」のみで証券なし → **保留**（証券提出まで承認不可）

### 2.3 労災条件（区分別）

| 区分 | 必須条件 |
|------|----------|
| **法人** | 労働者災害補償保険（労災保険）に加入済みであること。被用者を雇用する場合は事業所全体の加入を確認 |
| **個人事業主** | 一人親方労災特別加入 **または** 労災保険への加入（雇用形態に応じて）。建設・現場系業種では一人親方労災特別加入を原則とする |
| **一人親方** | **一人親方労災特別加入済み** を必須とする |
| **フリーランス** | 業種・作業内容により異なる（5章参照）。現場作業を伴う場合は一人親方労災特別加入または相当の補償を必須。リモートのみの IT・デザイン等は労災不要だが、保険・インボイスは必須 |

**労災未加入の扱い**

- 加入手続き中で証明が未取得 → **保留**
- 加入意思・手続きの意思がない → **否認**

### 2.4 承認時の最終判定ルール

```
IF 第9章チェックリスト全項目 = YES
   AND 保留理由が残存していない
   AND 否認条件に該当しない
THEN 承認可能
ELSE 承認不可（保留または否認）
```

# 3. 保留条件

### 3.1 保留になるケース

| コード | 保留理由 | 詳細 | 申請者への依頼内容 |
|--------|----------|------|-------------------|
| H01 | 添付書類不足 | 必要書類（第6章）の一部が未提出 | 不足書類の再提出依頼 |
| H02 | 保険証券未提出 | 加入申告はあるが証券画像・PDFなし | 保険証券の提出 |
| H03 | 保険内容要確認 | 証券提出済みだが補償額・被保険者名の確認が必要 | 補足資料または保険会社発行の証明 |
| H04 | インボイス番号確認中 | 国税庁照合結果待ち、または番号の入力ミス疑い | 正しい登録番号の再確認 |
| H05 | 実績確認中 | 申告実績の照合・参考先への確認中 | 施工写真・契約書・紹介先連絡先の追加提出 |
| H06 | 電話確認未実施 | 連絡先はあるが本人確認の架電が未完了 | （内部対応）担当者より架電 |
| H07 | 資格証確認待ち | 資格証の画像はあるが有効期限・記載名の確認中 | 資格証の鮮明な再提出または有効期限の補足 |
| H08 | 建設業許可確認中 | 許可番号の行政照合・有効性確認中 | 許可証写しの提出 |
| H09 | 反社チェック追加確認中 | 一次照合で要確認フラグ（将来API連携） | （内部対応）上長判断・追加調査 |
| H10 | 法人番号・登記不一致 | 法人番号と申告会社名の不一致 | 登記簿謄本または修正申請 |
| H11 | 対応エリア・業種の追加確認 | 申告内容と実態の乖離が疑われる | 対応可能範囲の説明・補足資料 |
| H12 | 労災加入証明待ち | 加入申告はあるが証明書未提出 | 労災加入証明の提出 |

### 3.2 保留解除条件

保留を解除し「審査待ち」または直接「承認」へ進めるには、以下を満たす。

| 条件 | 内容 |
|------|------|
| 保留理由の解消 | 該当する保留コード（H01〜H12）の不足・疑義がすべて解消されていること |
| 再確認完了 | 追加提出物について担当者が内容を確認し、備考に確認日・確認者を記録していること |
| チェックリスト更新 | 第9章の該当項目が YES に更新されていること |
| 申請者への連絡 | 保留解除・追加依頼の経緯をメール等で記録（Builder 備考または通知ログ） |

### 3.3 保留から否認への移行

以下のいずれかに該当した場合、保留から **否認** へ移行できる。

- 追加依頼から **30営業日** 経過しても提出・返信がない（連絡不能扱い）
- 追加提出内容に **虚偽** が判明した
- 保留理由の解消が不可能であることが確認された（例：許可の取得見込みなし）

# 4. 否認条件

### 4.1 否認になるケース

| コード | 否認理由 | 詳細 |
|--------|----------|------|
| R01 | 反社該当 | 反社DB・暴力団排除条例等に基づく照合で該当、または関係が確認された |
| R02 | 虚偽申告 | 故意による虚偽の申告（資格・実績・保険・労災・インボイス等） |
| R03 | インボイス未登録 | 適格請求書発行事業者登録がない、または失効・取消 |
| R04 | 保険未加入 | 加入意思がなく、かつ代替措置もない |
| R05 | 保険基準未達 | 対人1億円未満、または対物5,000万円未満で改善見込みなし |
| R06 | 労災未加入 | 区分・業種上必要な労災加入がなく、加入予定もない |
| R07 | 連絡不能 | 複数回の連絡試行（電話3回・メール2回以上）でも応答がない |
| R08 | 重大な行政処分歴 | 建設業法・労基法・消費者契約法等による **事業停止・許可取消** 等の重大処分 |
| R09 | 重大な過去トラブル歴 | 当社または関連事業における **重大なクレーム・未払い・安全上の重大事故** の記録 |
| R10 | 同意事項違反 | 申請時の確認・同意事項（反社表明・虚偽なし等）に違反が判明 |
| R11 | 重複・不正申請 | 同一事業体による不正な重複登録、なりすまし |
| R12 | 業種上の法的要件未達 | 許可・資格が法的に必須な業種で、取得見込みがない |

### 4.2 否認時の対応

- 申請者へ否認理由コードに基づく通知文を送付（個別事情に応じて担当者が調整）
- Builder 備考に否認理由コード・判断根拠・判断者・判断日を記録
- 個人情報はプライバシーポリシーに基づき適切に管理

### 4.3 否認後の再申請条件

| 否認理由 | 再申請 | 条件 |
|----------|--------|------|
| R03 インボイス未登録 | **可** | インボイス登録完了後、**3ヶ月経過後** から再申請可 |
| R04/R05 保険関連 | **可** | 保険加入・補償額基準達成後、証券提出とともに再申請可 |
| R06 労災未加入 | **可** | 労災加入完了後、証明提出とともに再申請可 |
| R07 連絡不能 | **可** | **6ヶ月経過後**、連絡可能な状態で再申請可 |
| R01 反社該当 | **不可** | 再申請不可 |
| R02 虚偽申告 | **不可** | 原則再申請不可。例外は管理者・法務承認が必要 |
| R08 重大行政処分 | **要審議** | 処分から **3年以上** 経過・改善実績がある場合に限り個別審議 |
| R09 重大トラブル歴 | **要審議** | 個別審議。和解・改善・再発防止策の提示が必要 |
| R11 不正申請 | **不可** | 再申請不可 |

**再申請時の扱い**

- 新規申請として受付し、過去の申請IDを備考に紐付け
- 否認理由コードと前回の審査結果を参照し、重点確認項目を設定

# 5. 業種別確認項目

登録フォームの業種選択に基づき、審査時に確認する推奨資格・実績・注意事項を定める。
**「推奨」は承認必須ではない** が、未保有の場合は実績確認・面談を強化する。
**法的に必須** の資格・許可がある業種はその旨を明記し、未達の場合は **否認（R12）** または **保留** とする。

---

### 5.1 電気工事

| 項目 | 内容 |
|------|------|
| 推奨資格 | 第一種または第二種電気工事士、1種または2種電気主任技術者（高圧・特高は別途） |
| 推奨実績 | 一般電気設備工事、分電盤交換、コンセント増設、LED化工事 等 **2件以上** の施工実績 |
| 注意事項 | 電気工事士法に基づく作業区分の遵守。無資格者による電気工事の請負は不可。建設業許可（電気工事業）の要否を確認 |

---

### 5.2 設備工事

| 項目 | 内容 |
|------|------|
| 推奨資格 | 給水装置工事主任技術者、ガス可とう管（該当作業時）、関連技能士 |
| 推奨実績 | 給排水・衛生設備、ボイラー・ポンプ設置 等の実績 |
| 注意事項 | ガス工事はガス事業法に基づく資格必須。水道工事は給水装置工事主任技術者が必要な場合あり |

---

### 5.3 空調工事

| 項目 | 内容 |
|------|------|
| 推奨資格 | 冷媒フロン類取扱技術者（第一種・第二種）、電気工事士（配線作業時） |
| 推奨実績 | エアコン設置・更新、換気設備工事 等 |
| 注意事項 | フロン類の取扱いには資格が法的に必要。高所作業・吊り作業の安全管理体制を確認 |

---

### 5.4 大工工事

| 項目 | 内容 |
|------|------|
| 推奨資格 | 建設業許可（木工事業等）、技能士（建築大工等）※資格は必須ではないが推奨 |
| 推奨実績 | 内装大工、造作工事、リフォーム大工事 等 **3件以上** |
| 注意事項 | 一人親方の場合は労災特別加入必須。足場・高所作業の安全対策を確認 |

---

### 5.5 塗装工事

| 項目 | 内容 |
|------|------|
| 推奨資格 | 建設業許可（塗装工事業）、有機溶剤作業主任者（該当作業時） |
| 推奨実績 | 外壁塗装、屋根塗装、内部塗装 等 |
| 注意事項 | 有機溶剤・鉛・PCB 等の特殊塗装は追加資格・体制が必要。飛散防止・近隣対応の実績を確認 |

---

### 5.6 防水工事

| 項目 | 内容 |
|------|------|
| 推奨資格 | 建設業許可（防水工事業）、防水施工技能士 ※技能士は推奨 |
| 推奨実績 | 屋上・ベランダ・外壁防水 等 **2件以上** |
| 注意事項 | 保証内容・施工年数の申告と実績の整合を確認。下地調査能力の有無を面談で確認 |

---

### 5.7 外構工事

| 項目 | 内容 |
|------|------|

# 7. 審査フロー

### 7.1 標準フロー

```
① 登録申請（申請者）
    │  Webフォーム送信（source: iwasho / tasful / builder）
    ↓
② 一次確認（審査担当）
    │  申請内容の欠損・明らかな矛盾のチェック
    │  → 問題あり：保留（H01等） / 問題なし：次へ
    ↓
③ 反社チェック（審査担当 / 将来API）
    │  反社DB照合 + 担当者確認
    │  → 該当：否認（R01） / 要確認：保留（H09） / 問題なし：次へ
    ↓
④ 保険確認（審査担当）
    │  証券提出・対人1億円以上・対物5,000万円以上・有効期限
    │  → 未提出：保留（H02） / 未達：否認（R05） / OK：次へ
    ↓
⑤ 労災確認（審査担当）
    │  区分別加入要件の確認
    │  → 未加入：保留（H12）または否認（R06） / OK：次へ
    ↓
⑥ インボイス確認（審査担当）
    │  国税庁公表サイト照合
    │  → 未登録：否認（R03） / 確認中：保留（H04） / OK：次へ
    ↓
⑦ 実績確認（審査担当）
    │  申告実績・業種別基準（第5章）・参考先確認
    │  → 確認中：保留（H05） / OK：次へ
    ↓
⑧ 連絡先・実在確認（審査担当）
    │  電話またはメールで意思確認・実在確認
    │  → 未実施：保留（H06） / 連絡不能：否認（R07） / OK：次へ
    ↓
⑨ 承認判断（審査担当）
    │  第9章チェックリスト全YES → ステータス「承認」
    ↓
⑩ 電子契約（申請者 + 運営）
    │  電子契約サービスで協力パートナー契約を締結（将来連携）
    ↓
⑪ 契約済み（システム / 担当者）
    │  ステータス「契約済み」に更新
    ↓
⑫ 案件紹介開始
       Builder・TASFUL・IWASHO 各チャネルでマッチング対象として有効化
```

### 7.2 フロー上の分岐ルール

| 状況 | 対応 |
|------|------|
| 複数の保留理由が同時に存在 | すべて解消するまで「保留」を維持 |
| 承認必須条件のいずれかが未達 | 「承認」不可。保留または否認を選択 |
| 電子契約が完了しないまま90日経過 | ステータスは「承認」のまま。備考に契約督促記録。案件紹介は不可 |
| IWASHO / TASFUL 流入元による差異 | 審査基準は共通。契約書テンプレート・紹介チャネルのみ流入元で分岐 |

### 7.3 審査 SLA（目安）

| 段階 | 目標日数（営業日） |
|------|-------------------|
| 登録申請 → 一次確認完了 | 2日以内 |
| 一次確認 → 承認または保留/否認判断 | 10日以内 |
| 保留 → 保留解除または否認 | 30日以内（申請者対応待ち除く） |
| 承認 → 電子契約案内 | 3日以内 |
| 電子契約締結 → 契約済み更新 | 契約完了即日 |

# 8. Builder連携前提

Builder 管理画面（`/builder/partner-management.html` および将来の本番DB）で管理する項目を以下に定義する。
審査基準との対応関係を併記し、システム実装時のスキーマ設計に利用する。

### 8.1 管理項目定義

| 項目名 | 型・形式 | 必須 | 説明 | 審査基準との対応 |
|--------|----------|------|------|------------------|
| **受付日** | date (YYYY-MM-DD) | ○ | 登録申請の受付日時（日付部分） | フロー① |
| **流入元** | enum: `iwasho` / `tasful` / `builder` | ○ | 登録フォームの hidden `source` | 契約テンプレ・紹介先の分岐 |
| **会社名** | string | ○ | 会社名・屋号 | 実在確認・保険被保険者照合 |
| **区分** | enum: 法人 / 個人事業主 / 一人親方 / フリーランス | ○ | 事業形態 | 労災要件（2.3）、書類（第6章） |
| **業種** | string[] (複数) | ○ | 登録フォームの業種チェック値 | 第5章の個別確認 |
| **対応エリア** | string | ○ | 対応可能エリア | マッチング・紹介時の絞り込み |
| **インボイス** | string (T+13桁) | ○ | 適格請求書発行事業者登録番号 | 承認必須条件②、否認R03 |
| **保険** | enum: 加入済み / 未加入 / 加入予定 / 確認中 | ○ | 保険加入状況（概要） | 承認必須条件③、2.2 |
| **労災** | enum: 法人労災 / 一人親方労災 / 未加入 / 確認中 | ○ | 労災加入状況（概要） | 承認必須条件④、2.3 |
| **反社** | enum: 未確認 / 問題なし / 要確認 / 該当 | ○ | 反社チェック結果 | 承認必須条件①、否認R01 |
| **ステータス** | enum: pending / hold / approved / rejected / contracted | ○ | 審査ステータス | 第1章 |
| **契約状況** | enum: 未契約 / 契約案内済 / 契約締結済 / 契約解除 | ○ | 電子契約の進捗 | 契約済みステータスとの連動 |
| **備考** | text | — | 審査メモ・保留/否認理由・変更履歴 | 1.4、3、4章の記録義務 |

### 8.2 将来拡張項目（推奨）

| 項目名 | 説明 |
|--------|------|
| 申請ID | `PR-YYYY-NNN` 形式の一意識別子 |
| 代表者名 / 担当者名 / 電話 / メール | 連絡先確認用 |
| 保険_対人補償額 / 保険_対物補償額 | 数値基準判定用 |
| 保留理由コード | H01〜H12 |
| 否認理由コード | R01〜R12 |
| チェックリストJSON | 第9章の YES/NO 状態 |
| 電子契約ID | 電子契約サービス連携用 |
| 審査担当者ID | 担当者の記録 |
| 最終更新日時 | 監査用 |

### 8.3 ステータスと契約状況の連動ルール

| ステータス | 契約状況（想定） | 案件紹介 |

| rejected | — | 不可 |

# 9. 審査担当チェックリスト

審査担当者は以下を **YES / NO** で記録し、**すべて YES の場合のみ承認** できる。
Builder 実装時はチェックボックスまたはトグルとして UI 化する。

### 9.1 必須チェック項目

| # | チェック項目 | YES の条件 | 対応する承認必須条件 |
|---|-------------|-----------|---------------------|
| 1 | □ 反社チェック完了 | 反社該当なし・要確認事項なし | 反社チェック問題なし |
| 2 | □ インボイス確認完了 | 国税庁照合で有効な登録番号を確認 | インボイス登録済み |
| 3 | □ 保険確認完了 | 証券提出済み、対人≧1億円・対物≧5,000万円、有効期限内 | 保険加入済み |
| 4 | □ 労災確認完了 | 区分・業種に応じた労災加入を確認 | 労災加入済み |
| 5 | □ 連絡先確認完了 | 電話またはメールで申請者・担当者と意思確認済み | 連絡先確認済み |
| 6 | □ 実在確認完了 | 登記・開業・Web等で事業体の実在を確認 | 実在確認済み |
| 7 | □ 必要書類確認完了 | 第6章の区分・業種に応じた書類が揃っている | （総合） |
| 8 | □ 実績確認完了 | 申告実績が業種基準（第5章）を満たす、または面談で代替確認 | （業種別） |
| 9 | □ 業種別要件確認完了 | 法的必須の許可・資格がある場合、それを確認済み | 否認R12の回避 |
| 10 | □ 同意事項確認完了 | 申請時の確認・同意チェックに矛盾がない | 否認R10の回避 |

### 9.2 承認判定

```
承認可能 = (項目1〜10 すべて YES) AND (ステータスが保留理由を持たない)
```

### 9.3 チェックリスト記録例（備考欄）

```
[審査チェックリスト] 2026-06-23 担当: 山田
1.反社: YES  2.インボイス: YES  3.保険: YES  4.労災: YES
5.連絡先: YES  6.実在: YES  7.書類: YES  8.実績: YES
9.業種要件: YES  10.同意: YES
→ 承認
```

### 9.4 保留・否認時のチェックリスト

- 保留時: NO の項目に対応する保留コード（H01〜H12）を備考に記載
- 否認時: NO の項目に対応する否認コード（R01〜R12）を備考に記載
- 再申請時: 前回 NO だった項目を重点確認
