# 1. 文書概要

## 1.1 目的

本フローは、協力パートナーの **登録申請から電子契約締結・継続運用** までの業務を標準化し、以下を実現する。

- 契約締結業務の標準化と担当者間の判断統一
- 契約書・署名・関連書類の **証跡保管**
- 保険・労災・インボイス・許可証の **更新管理**
- **案件紹介開始条件** の明確化（契約済みかつ有効状態のみ）

## 1.2 対象

| 区分 | 説明 |
|------|------|
| **法人** | 株式会社・合同会社等 |
| **個人事業主** | 開業届を提出している個人事業 |
| **一人親方** | 個人で現場作業を行う事業者 |
| **フリーランス** | 個人でサービスを提供する事業者 |

## 1.3 対象契約

| 契約書名称 | 適用 |
|-----------|------|
| **IWASHO協力パートナー基本契約書** | 流入元 `iwasho` の協力パートナー |
| **TASFUL協力パートナー基本契約書** | 流入元 `tasful` の協力パートナー（名称・条項はIWASHO版と同等の運用ルールを適用） |
| **Builder経由登録** | 流入元 `builder` の場合、最終的な紹介チャネルに応じて IWASHO または TASFUL 契約書を適用 |

**共通条項（すべての契約書に含める）**

- 反社会的勢力排除条項（`partner-anti-social-checklist.md` 第8章と整合）
- インボイス・保険・労災に関する表明・確約
- 虚偽申告時の契約解除条項
- 個人情報・機密情報の取扱い

## 1.4 本フローの位置付け

本フローは、審査基準における **「承認（approved）」以降〜「契約済み（contracted）」以降の継続運用** を担う。
審査・反社チェック完了前の契約送付は **禁止** とする。

# 2. 全体フロー

## 2.1 エンドツーエンドフロー

```
① 登録申請（申請者）
    │  Webフォーム送信
    ↓
② 一次確認（審査担当）
    │  申請内容の欠損・矛盾チェック
    ↓
③ 反社チェック（審査担当）
    │  反社チェックシートに基づく確認
    │  → 該当：否認（R01） / 要確認：保留（H09）
    ↓
④ 審査承認（審査担当 + 運営管理者）
    │  審査基準 第9章チェックリスト全YES
    │  ステータス「承認（approved）」
    ↓
⑤ 契約書送付（契約担当）
    │  第3章の送付条件をすべて満たすこと
    │  電子契約サービスで契約書を送信
    ↓
⑥ 電子署名（申請者）
    │  契約内容の確認・電子署名
    ↓
⑦ 契約締結（システム / 契約担当）
    │  署名完了を確認
    │  ステータス「契約済み（contracted）」
    ↓
⑧ 案件紹介開始（運営 / マッチング担当）
    │  契約済みかつ契約状態「有効」の場合のみ
```

## 2.2 工程別担当者

| 工程 | 主担当 | 副担当・承認 |
|------|--------|-------------|
| 登録申請 | 申請者（協力パートナー） | — |
| 一次確認 | 審査担当者 | — |
| 反社チェック | 審査担当者 | 要確認・該当時は運営管理者 |
| 審査承認 | 審査担当者 | 運営管理者（承認判断の最終確認） |
| 契約書送付 | 契約担当者 | 審査担当者（送付前条件確認） |
| 電子署名 | 申請者（代表者または authorized 担当者） | — |
| 契約締結確認 | 契約担当者 | — |
| 案件紹介開始 | マッチング担当 / 運営 | 契約担当者（有効状態の確認） |
| 更新管理 | 契約担当者 | 審査担当者（書類再確認） |
| 契約停止・解除 | 運営管理者 | 法務担当（重大事由時） |

## 2.3 ステータスとの対応

| 審査ステータス | 本フロー上の位置 | 案件紹介 |
|---------------|-----------------|----------|
| `pending` / `hold` | ②〜④の途中 | 不可 |
| `approved` | ⑤〜⑦の途中 | 不可 |
| `rejected` | フロー終了（否認） | 不可 |
| `contracted` | ⑧以降 | **可**（契約状態が有効の場合） |

# 3. 契約送付条件

## 3.1 送付可能条件（すべて必須）

以下 **6条件をすべて満たす場合のみ** 契約書を送付できる。
**1つでも未完了の場合は送付不可** とする。

| # | 条件 | 確認根拠 | 審査基準との対応 |
|---|------|----------|-----------------|
| 1 | **審査承認済み** | ステータス = `approved` | 第1章・第9章 |
| 2 | **反社チェック問題なし** | 反社判定 = `clear`（問題なし） | 反社チェックシート 第2章 |
| 3 | **インボイス確認済み** | 国税庁照合で有効な T+13桁 | 承認必須条件② |
| 4 | **保険確認済み** | 証券提出済み、対人≧1億円・対物≧5,000万円、有効期限内 | 承認必須条件③ |
| 5 | **労災確認済み** | 区分・業種に応じた加入確認済み | 承認必須条件④ |
| 6 | **必要書類確認済み** | 審査基準 第6章の区分・業種別書類が揃っている | チェックリスト項目7 |

## 3.2 送付不可時の対応

| 未達条件 | 対応 |
|----------|------|
| 審査未承認 | 審査継続。保留・否認の場合は契約フローに入らない |
| 反社要確認 | 保留（H09）のまま。反社チェック完了まで送付不可 |
| 反社該当 | 否認（R01）。送付不可 |
| インボイス・保険・労災・書類 | 保留のまま追加提出を依頼 |

## 3.3 送付前の最終確認（契約担当者）

契約送付ボタン操作前に、第12章チェックリスト項目1〜5がすべて YES であることを確認する。
送付実施後、Builder に **契約送付日・送付担当者** を記録する。

# 4. 電子契約フロー

## 4.1 標準フロー（手動運用 / 将来API連携）

```
① 契約書生成
    │  テンプレート（IWASHO / TASFUL）に申請情報を反映
    │  当事者名・住所・代表者・業種・インボイス番号等を自動または手動入力
    ↓
② 送信
    │  申請者のメールアドレス宛に電子契約サービスから送付
    │  Builder「契約状態」→ 送付済み
    ↓
③ 相手確認
    │  申請者が契約書を閲覧
    │  Builder「契約状態」→ 閲覧済み（サービス連携時）
    ↓
④ 電子署名
    │  申請者（原則：代表者または申告担当者）が署名
    │  Builder「契約状態」→ 署名待ち → 完了
    ↓
⑤ 完了通知
    │  当社側にも署名完了通知
    │  契約担当者が内容を最終確認
    ↓
⑥ 契約保管
    │  署名済みPDFを証跡ストレージに保管
    │  ステータス「契約済み（contracted）」に更新
```

## 4.2 記録項目（必須）

| 項目 | 形式 | 記録タイミング | 記録先 |
|------|------|---------------|--------|
| **契約送付日** | date (YYYY-MM-DD) | 送信時 | Builder |
| **送付担当者** | string | 送信時 | Builder |
| **締結日** | date (YYYY-MM-DD) | 署名完了時 | Builder |
| **契約番号** | string（例: `CNT-IW-2026-0001`） | 契約書生成時 | Builder |
| **保管場所** | string（ストレージパス / 電子契約サービスID） | 保管完了時 | Builder備考 + ストレージ |
| **電子契約ID** | string | サービス連携時 | Builder（将来） |
| **契約書種別** | enum: `iwasho` / `tasful` | 生成時 | Builder |

### 契約番号の採番ルール（推奨）

```
CNT-{流入元略称}-{YYYY}-{連番4桁}

例:
CNT-IW-2026-0001  （IWASHO）
CNT-TF-2026-0042  （TASFUL）
```

## 4.3 署名者の要件

| 区分 | 署名者 |
|------|--------|
| 法人 | 代表者、または委任状に基づく担当者 |
| 個人事業主 / 一人親方 / フリーランス | 申請者本人 |

委任による署名の場合は、委任状を証跡として保管する。

# 5. 契約未締結管理

## 5.1 契約進捗状態（契約状態の詳細）

審査基準の「契約状況」を、電子契約運用向けに以下の **6状態** で管理する。

| 状態 | 内部コード | 意味 |
|------|-----------|------|
| **未送付** | `not_sent` | 審査承認済みだが契約書未送付 |
| **送付済み** | `sent` | 電子契約サービスから送付完了。相手未閲覧または未署名 |
| **閲覧済み** | `viewed` | 申請者が契約書を閲覧済み（サービス連携時） |
| **署名待ち** | `awaiting_signature` | 申請者側の署名待ち、または当社側署名待ち |
| **完了** | `completed` | 双方の署名完了。契約締結済み |
| **辞退** | `declined` | 申請者が契約を辞退、または期限切れ失効 |

### 審査ステータス・契約状況との対応

| 契約進捗状態 | 審査ステータス | 契約状況（審査基準） | 案件紹介 |
|-------------|---------------|---------------------|----------|
| 未送付 | `approved` | 未契約 / 契約案内済 | 不可 |
| 送付済み / 閲覧済み / 署名待ち | `approved` | 契約案内済 | 不可 |
| 完了 | `contracted` | 契約締結済 | **可** |
| 辞退 | `approved` または `rejected` | 未契約 / 契約解除 | 不可 |

## 5.2 催促ルール

| 経過日数 | アクション | 担当 | Builder記録 |
|----------|-----------|------|------------|
| 承認後 **3営業日** | 契約書送付（SLA） | 契約担当 | 送付日 |
| 送付後 **7日** | **第1回リマインド**（メール） | 契約担当 | 備考に催促日 |
| 送付後 **14日** | **第2回リマインド**（メール + 電話） | 契約担当 | 備考に催促日 |
| 送付後 **30日** | **契約失効**（`declined`） | 契約担当 + 運営管理者 | 備考に失効理由 |
| 失効後 | ステータスは `approved` 維持または個別判断で `rejected` へ | 運営管理者 | 再送付要否を判断 |

**失効後の再送付**

- 申請者からの依頼があり、審査内容に変更がなければ **再送付可**
- 保険・労災・インボイスの有効期限が切れている場合は **再確認後** に再送付

## 5.3 承認後90日ルール（審査基準との整合）

審査基準第7章に基づき、承認後 **90日** 経過しても契約締結がない場合：

- ステータスは `approved` のまま維持可能
- 備考に契約督促・失効の記録を残す
- **案件紹介は引き続き不可**
- 必要に応じて審査内容の **再確認**（保険・労災・インボイス・反社）を実施

# 6. 契約後管理

## 6.1 案件紹介開始条件

**以下をすべて満たす場合のみ、案件紹介・マッチングが可能** とする。

| # | 条件 |
|---|------|
| 1 | 審査ステータス = `contracted`（契約済み） |
| 2 | 契約進捗状態 = `completed`（署名完了） |
| 3 | 契約状態（有効性）= `active`（有効） |
| 4 | 反社判定 = `clear`（問題なし） |
| 5 | 保険が有効期限内 |
| 6 | 労災加入が有効 |
| 7 | インボイス登録が有効（取消・失効なし） |

**いずれか1つでも未達の場合、案件紹介は停止する。**

## 6.2 契約後管理項目

| 管理項目 | 説明 | 更新頻度 |
|----------|------|----------|
| **契約締結日** | 電子署名完了日 | 締結時 |
| **契約状態** | 有効 / 停止 / 解除（第9章） | 変更時 |
| **保険期限** | 保険証券の満了日 | 更新時 |
| **労災期限** | 労災加入の有効期限・更新日 | 更新時 |
| **インボイス状態** | 有効 / 取消 / 失効 / 確認中 | 月次または変更時 |
| **許可証期限** | 建設業許可等の有効期限 | 更新時 |
| **最終確認日** | 運営による一括確認の最終実施日 | 年次 |

## 6.3 契約後の定期確認（年次）

契約締結後、**毎年1回** 以下を確認する（第7章と連動）。

- 保険証券の有効性
- 労災加入の継続
- インボイス登録状態
- 許可証・資格証の有効性
- 反社チェック（変更があった場合）

# 7. 更新管理

## 7.1 毎年確認対象

| 対象 | 確認内容 | 未更新時の影響 |
|------|----------|---------------|
| **保険** | 証券の有効期限、対人≧1億円・対物≧5,000万円の維持 | 案件紹介停止 → 契約停止検討 |
| **労災** | 加入の継続、証明の有効性 | 案件紹介停止 → 契約停止検討 |
| **許可証** | 建設業許可等の有効期限 | 該当業種の案件紹介停止 |
| **資格証** | 電気工事士等の有効期限 | 該当作業の案件紹介停止 |

## 7.2 更新フロー

```
① 期限前通知（契約担当 → 協力パートナー）
    │  満了日の 60日前 / 30日前 / 14日前 に通知（推奨）
    ↓
② 再提出（協力パートナー）
    │  更新後の保険証券・労災証明・許可証・資格証を提出
    ↓
③ 確認（審査担当 / 契約担当）
    │  審査基準に基づき内容を再確認
    │  問題なし → Builder の期限日を更新
    ↓
④ 更新完了
    │  契約状態を「有効」に維持
    │  案件紹介を継続可
```

## 7.3 期限切れ時の対応

| 経過 | 対応 |
|------|------|
| 期限切れ当日 | 案件紹介 **即時停止**。協力パートナーへ更新依頼 |
| 期限切れ後14日 | 契約状態を **停止** に変更（第9章） |
| 期限切れ後30日 | 運営管理者判断で **解除** 検討 |
| 更新提出・確認完了 | 停止解除、案件紹介再開 |

## 7.4 更新記録

Builder 備考または専用フィールドに以下を記録する。

- 更新対象（保険 / 労災 / 許可 / 資格）
- 提出日・確認日・確認者
- 新しい有効期限
- 判定（更新完了 / 保留 / 停止）

# 8. インボイス管理

## 8.1 確認事項

| 項目 | 確認方法 | 頻度 |
|------|----------|------|
| **登録番号** | 申告の T+13桁と国税庁公表サイトの一致 | 初回審査時・年次・変更時 |
| **登録状態** | 公表サイトで「登録済み」であること | 月次一括照合（推奨）または年次 |
| **取消・失効の有無** | 公表サイトの状態変更を監視 | 月次一括照合（推奨） |

## 8.2 インボイス状態の定義

| 状態 | 内部コード | 意味 | 案件紹介 |
|------|-----------|------|----------|
| 有効 | `valid` | 登録済み・取消なし | 可 |
| 確認中 | `checking` | 照合中・情報更新待ち | 停止 |
| 取消 | `cancelled` | 登録取消が確認された | 停止 |
| 失効 | `expired` | 登録失効が確認された | 停止 |

## 8.3 取消・失効時のルール

```
インボイス取消・失効を検知
    ↓
① 契約状態を「停止」に変更（即時）
    ↓
② 案件紹介を停止
    ↓
③ 協力パートナーへ再登録・状況確認を依頼
    ↓
④ 再確認
    ├→ 再登録完了・照合OK → 有効に戻す、案件紹介再開
    ├→ 確認中・未対応 → 停止維持（保留扱い）
    └→ 再登録なし・長期未対応 → 契約解除検討（第9章）
```

## 8.4 記録

- 検知日・検知方法（手動照合 / 将来API）
- 対応日・対応者
- 再登録後の新番号（ある場合）

# 9. 契約停止・解除管理

## 9.1 契約状態（有効性）

| 状態 | 内部コード | 意味 | 案件紹介 |
|------|-----------|------|----------|
| **有効** | `active` | 契約締結済みで停止事由なし | 可 |
| **停止** | `suspended` | 一時的に案件紹介・取引を停止 | 不可 |
| **解除** | `terminated` | 契約を終了 | 不可 |

**注記**: 審査ステータス `contracted` は維持し、契約状態で有効性を管理する（審査基準 第1章）。

## 9.2 停止事由

| コード | 停止事由 | 対応 |
|--------|----------|------|
| S01 | **虚偽申告** | 即時停止。調査後、解除（R02/R10） |
| S02 | **保険失効** | 更新提出まで停止 |
| S03 | **労災失効** | 更新提出まで停止 |
| S04 | **反社判明** | 即時停止。解除（R01） |
| S05 | **契約違反** | 内容に応じて停止・解除 |
| S06 | **顧客抜き** | 即時停止。解除検討 |
| S07 | **SNS違反** | 調査後、停止または解除 |
| S08 | **インボイス取消・失効** | 第8章に従い停止 |
| S09 | **許可証・資格の失効** | 該当業種の紹介停止。全体停止は個別判断 |
| S10 | **重大クレーム・安全事故** | 調査後、停止または解除 |

## 9.3 解除事由

以下の場合、契約状態を **解除（terminated）** とする。

- 反社該当が確定（R01）
- 虚偽申告が確定（R02/R10）
- 協力パートナーからの解約申出（書面または電子契約上の手続）
- 停止状態が **90日以上** 継続し、改善見込みがない
- 当社都合による契約終了（運営管理者承認）

## 9.4 停止・解除時の手続

1. 運営管理者または契約担当が停止・解除を決定
2. Builder に停止事由コード・日付・理由を記録
3. 協力パートナーへ通知（メール。重大事由は書面も検討）
4. 案件紹介・マッチングから即時除外
5. 証跡・契約書は第11章に従い保管継続

## 9.5 停止解除条件

| 事由 | 解除条件 |
|------|----------|
| S02/S03 保険・労災 | 有効な証明の再提出・確認完了 |
| S08 インボイス | 再登録確認完了 |
| S05/S07/S10 | 調査完了・改善策の提出・運営管理者承認 |
| S01/S04/S06 | 原則 **解除のみ**（停止解除不可） |

# 10. Builder連携

## 10.1 管理項目一覧

Builder 管理画面および将来の本番DBで、電子契約運用に関する項目を以下のとおり定義する。

| 項目名 | 型・形式 | 必須 | 説明 |
|--------|----------|------|------|
| **契約番号** | string | ○ | `CNT-IW-YYYY-NNNN` 形式 |
| **契約状態** | enum: `active` / `suspended` / `terminated` | ○ | 有効 / 停止 / 解除 |
| **契約進捗** | enum: `not_sent` / `sent` / `viewed` / `awaiting_signature` / `completed` / `declined` | ○ | 第5章 |
| **送付日** | date | 送付後必須 | 契約書送付日 |
| **締結日** | date | 締結後必須 | 電子署名完了日 |
| **更新期限** | date | 推奨 | 次回年次確認または契約更新の目安日 |
| **保険期限** | date | ○ | 保険証券満了日 |
| **労災期限** | date | 推奨 | 労災加入の確認・更新期限 |
| **インボイス状態** | enum: `valid` / `checking` / `cancelled` / `expired` | ○ | 第8章 |
| **送付担当者** | string | 送付後必須 | 契約送付を実施した担当者 |
| **電子契約ID** | string | 将来 | 電子契約サービス連携用 |
| **保管場所** | string | 締結後必須 | 署名済み契約書の保管先 |

### 審査基準フィールドとの統合

| 審査基準（既存） | 本フロー（拡張） |
|-----------------|-----------------|
| ステータス `contracted` | 契約進捗 = `completed` かつ 契約状態 = `active` |
| 契約状況（未契約/案内済/締結済/解除） | 契約進捗 + 契約状態で詳細管理 |
| 備考 | 催促・失効・停止・更新の履歴 |

## 10.2 ステータス連動ルール

| 審査ステータス | 契約進捗 | 契約状態 | 案件紹介 | 自動更新トリガー |
|---------------|----------|----------|----------|-----------------|
| `approved` | `not_sent`〜`awaiting_signature` | — | 不可 | 承認時に契約進捗=未送付 |
| `approved` | `declined` | — | 不可 | 30日失効時 |
| `contracted` | `completed` | `active` | **可** | 署名完了時 |
| `contracted` | `completed` | `suspended` | 不可 | 停止事由発生時 |
| `contracted` | `completed` | `terminated` | 不可 | 解除時 |
| `rejected` | — | — | 不可 | 否認時 |

## 10.3 連動更新の手順（手動運用）

| イベント | Builder更新内容 |
|----------|----------------|
| 審査承認 | ステータス=`approved`、契約進捗=`not_sent` |
| 契約送付 | 送付日、送付担当者、契約進捗=`sent` |
| 署名完了 | 締結日、契約進捗=`completed`、ステータス=`contracted`、契約状態=`active` |
| 保険期限切れ | 契約状態=`suspended`、停止事由=S02 |
| インボイス取消 | インボイス状態=`cancelled`、契約状態=`suspended`、停止事由=S08 |
| 契約解除 | 契約状態=`terminated`、契約状況=契約解除 |

# 11. 証跡保管

## 11.1 保管対象

| 資料 | 保管タイミング | 形式 |
|------|---------------|------|
| **契約書**（署名済み） | 締結完了時 | PDF（電子契約サービスから取得） |
| **保険証券** | 初回審査時・更新時 | PDF / 画像 |
| **労災加入証明** | 初回審査時・更新時 | PDF / 画像 |
| **資格証** | 初回審査時・更新時 | PDF / 画像 |
| **建設業許可証** | 該当業種・初回・更新時 | PDF / 画像 |
| **反社チェック記録** | 審査時 | 電子データ（反社チェックシート写し） |
| **催促・通知メール** | 送付時 | ログまたはスクリーンショット |

## 11.2 推奨保管期間

| 資料 | 保管期間 |
|------|----------|
| すべての証跡 | **契約期間中** + **契約終了後5年間** |
| 否認・解除関連 | 契約終了後5年間（説明責任・再申請防止） |

反社チェックシート第13章・審査基準と同一方針とする。

## 11.3 保管場所・権限

| 項目 | 方針 |
|------|------|
| 一次保管 | 電子契約サービス + 運営用ストレージ |
| Builder | 契約番号・保管場所（パスまたはID）を記録。ファイル本体は権限分離 |
| アクセス権限 | 契約担当・審査担当・運営管理者のみ |
| バックアップ | 年次バックアップを推奨 |

## 11.4 個人情報保護

- 保管データはプライバシーポリシーに基づき管理
- 契約終了後5年経過した資料は、法令で longer の保存が求められない限り削除または匿名化

# 12. 運用担当チェックリスト

契約送付〜案件紹介開始まで、運用担当者が **YES / NO** で記録するチェックリスト。
**すべて YES の場合のみ案件紹介可能** とする。

## 12.1 契約送付前チェック（項目1〜5）

| # | チェック項目 | YES の条件 |
|---|-------------|-----------|
| 1 | □ **審査承認済み** | ステータス = `approved` |
| 2 | □ **反社問題なし** | 反社判定 = `clear` |
| 3 | □ **インボイス確認済み** | 国税庁照合で有効 |
| 4 | □ **保険確認済み** | 証券提出済み、基準額・有効期限内 |
| 5 | □ **労災確認済み** | 区分・業種に応じた加入確認済み |

## 12.2 契約締結・案件紹介前チェック（項目6〜8）

| # | チェック項目 | YES の条件 |
|---|-------------|-----------|
| 6 | □ **契約送付済み** | 送付日記録あり、契約進捗 ≧ `sent` |
| 7 | □ **電子署名完了** | 締結日記録あり、契約進捗 = `completed` |
| 8 | □ **証跡保存済み** | 署名済み契約書PDFが保管場所に記録されている |

## 12.3 案件紹介開始の最終判定

```
案件紹介可能 =
  (項目1〜8 すべて YES)
  AND ステータス = contracted
  AND 契約状態 = active
  AND 保険・労災・インボイスが有効
```

## 12.4 記録例（Builder備考）

```
[契約運用チェック] 2026-06-23 担当: 田中
1.審査承認: YES  2.反社: YES  3.インボイス: YES
4.保険: YES  5.労災: YES  6.送付: YES  7.署名: YES  8.証跡: YES
契約番号: CNT-IW-2026-0007
→ 案件紹介開始可
```

# 13. 将来拡張

以下は現時点では手動運用とし、将来のシステム連携時に追加する機能として整理する。

| 機能 | 概要 | 連携先 |
|------|------|--------|
| **電子契約サービス連携** | 契約書生成・送付・署名・保管の API 自動化 | CloudSign / DocuSign / クラウドサイン等 |
| **自動リマインド** | 送付後7日・14日のメール自動送信 | メール通知基盤 / Builder |
| **保険期限通知** | 満了60/30/14日前の自動通知 | Builder + メール |
| **労災期限通知** | 同上 | Builder + メール |
| **インボイス自動照会** | 国税庁APIまたは一括照合バッチ | Builder インボイス状態の自動更新 |
| **Builder自動更新管理** | 期限切れ時の自動停止、更新提出時の自動解除 | Builder + ワークフロー |
| **反社定期再スクリーニング** | 契約中パートナーの定期照会 | 反社API + Builder |
| **契約更新の電子化** | 覚書・更新契約の電子締結 | 電子契約サービス |

## 13.1 拡張時のデータ設計方針

- 本フロー第10章の管理項目を DB スキーマの基準とする
- イベント駆動（署名完了 Webhook → ステータス `contracted` 自動更新）を想定
- 手動運用で記録している項目は、すべてシステム化後も維持する
