## Backlog（将来カテゴリ）— ミニゲーム / ブラウザゲーム

**MVP 対象外** · 素材DL本体とは **別 surface 候補**（例: `games/`）。TASFUL 傘下で将来検討。

### ゲーム種別

| 種別 | 備考 |
| --- | --- |
| HTML5ミニゲーム | Canvas/WebGL · 単体 HTML またはバンドル |
| 昔のフラッシュゲーム風コンテンツ | **Flash 本体は使わない** · HTML5/JS で再現 |
| パズル | 短時間プレイ · 広告挿入ポイント設計 |
| 脱出ゲーム | シーン遷移 · タップ/クリック UI |
| タイピングゲーム | 日本語 IME · スコア表示（ランキングは後回し） |
| 子ども向け知育ゲーム | COPPA/年齢表示 · 広告制限要検討 |
| 暇つぶしゲーム | セッション短 · モバイルファースト |
| 野球・スポーツ系ミニゲーム | TLV とは **別** · 権利・実在チーム名に注意 |

### 収益モデル（案）

| モデル | 内容 |
| --- | --- |
| リワード広告 | 続行/ヒント/追加ライフ等 |
| インタースティシャル広告 | ステージ間 · 頻度上限要設計 |
| 広告なし Premium | サブスク or 単品 · Materials Premium とバンドル検討可 |

### 注意事項

| 項目 | 方針 |
| --- | --- |
| 技術 | Flash 非使用 · HTML5/JavaScript のみ |
| 権利 | 著作権 · キャラクター · BGM · 効果音の権利確認（Materials 素材流用時もライセンス整合） |
| 端末 | **スマホ対応必須**（390/768 検証 · タッチ操作） |
| 後回し | セーブデータ · ランキング · アカウント連携 · マルチプレイ |

**関連:** TLV は FEATURE FROZEN（AD-004）— ミニゲーム surface は TLV 統合しない

## Backlog（将来追加判断）

以下は **MVP 着手前に固定しない**。Materials MVP 稼働後、**需要 · 収益 · アクセス解析** を見て追加可否を判断する。

### 判断ゲート（案）

| 指標 | 判断に使う例 |
| --- | --- |
| **需要** | カテゴリ検索数 · DL 数 · 離脱率 · 関連クエリ（サイト内検索ログ） |
| **収益** | 広告 RPM/eCPM · Premium 転換 · カテゴリ別 ARPU · 運用コスト（生成/審査/Storage） |
| **アクセス解析** | PV/セッション · 流入元 · デバイス比率 · リピート率 · カテゴリ横断率 |

**Go 条件（例）:** 3 ヶ月連続でカテゴリ需要上位 · 収益が運用コストを上回る見込み · 法務/ライセンス整理済み  
**No-Go / 延期:** 需要薄 · 収益化不明 · 権利リスク高 · 既存製品（Builder/AI/TLV）と責務が競合

### 追加判断リスト

| 候補 | 位置づけ | メモ |
| --- | --- | --- |
| **テンプレートライブラリ拡張** | 本 Backlog § テンプレートライブラリ | MVP は SNS/サムネのみ · 名刺/Office/PDF 等は需要次第 |
| **ブラウザゲーム** | 本 Backlog § ミニゲーム | 別 surface（`games/`）· 広告モデル別設計 |
| **独自AI** | TASFUL AI / Gateway 連携候補 | **TASFUL AI Workspace 統合は別判断**（AD-002/005）· Materials 向け軽量提案 AI の可否 |
| **レシピ** | 無料コンテンツ候補 | テキスト+画像 · 印刷/PDF · 栄養/アレルギー表記要検討 |
| **学習コンテンツ** | 無料コンテンツ候補 | 教材/クイズ/解説 · 年齢区分 · 広告/子ども向け制限 |
| **その他無料コンテンツ** | 汎用バケット | 需要上位の未カバーカテゴリを PoC 後にスロットイン |

### 計測の前提（MVP 時点で入れておく）

- カテゴリ · 素材 ID · DL イベント · 検索クエリの **匿名集計**
- 広告インプレッション/クリック（カテゴリ・ページ種別）
- **個人を特定しない** ダッシュボード（運営判断用）
