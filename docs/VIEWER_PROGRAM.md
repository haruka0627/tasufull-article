# Live Platform — 視聴者制度（VIEWER PROGRAM）

**最終更新:** 2026-06-28  
**種別:** 制度設計正本（**実装なし**）  
**親:** [LIVE_PLATFORM_CONCEPT.md](./LIVE_PLATFORM_CONCEPT.md)  
**AD:** [DECISIONS.md](./DECISIONS.md) **AD-014**（Viewer Program 採用）

---

## 1. 目的

視聴者が **楽しみ · 応援し · 長く居続ける** ための参加型制度。  
クリエイター収益（[MONETIZATION.md](./MONETIZATION.md)）と両立する。

**原則（AD-012）:** 高機能は AI · **UI はシンプル**。複雑な制度は段階解禁。

---

## 2. 採用（✅ Adopted）

| 制度 | 概要 |
| --- | --- |
| **視聴者レベル** | 視聴 · 応援 · コメント · 継続で XP — シンプル表示 |
| **古参バッジ** | アカウント作成 · チャンネル初視聴からの経過日数 |
| **サポーター** | チャンネル単位の応援表示 · メンバー相当（将来課金連動） |
| **ストリーク** | 連続視聴 / 連続ログイン · ライブ参加ストリーク |
| **ライブランキング** | 配信中の応援ランキング（ギフト · コメント · 視聴時間） |
| **応援（Cheer）** | ライブ中のリアクション · ゲージ連動（[LIVE_SYSTEM.md](./LIVE_SYSTEM.md)） |

---

## 3. 保留（⏸ Hold）

| 制度 | 概要 | 保留理由 |
| --- | --- | --- |
| **コレクション** | バッジ · 限定スタンプ · イベント記念品 | 経済設計 · 不正対策 |
| **マイル** | 視聴行動のポイント → 特典交換 | コスト · インフレ設計 |
| **ミッション** | デイリー / ウィークリー視聴タスク | ボット視聴リスク |
| **グローバルランキング** | 全プラットフォーム横断ランキング | 摩擦 · プライバシー |
| **サポーター特典詳細** | 限定バッジ · コメントハイライト | 課金基盤待ち |

---

## 4. 将来（📋 Future）

| 制度 | 概要 |
| --- | --- |
| **視聴者ギルド** | 同じクリエイターを応援するコミュニティ |
| **コラボ視聴イベント** | 複数チャンネル同時視聴 |
| **視聴者 Creator へのフィードバック** | 匿名アンケート · Q&A |
| **Family / ペア視聴** | 子供アカウント · ペアレンタル（法務後） |
| **Platform 横断プロフィール** | Platform お気に入りとの連携（AD 整合後） |

---

## 5. 不採用 / 禁止

| 項目 | 理由 |
| --- | --- |
| ギャンブル性の高いガチャ | 法務 · ブランド |
| 視聴者間 PvP 課金競争の過度な可視化 | 未成年 · 依存性 |
| 個人の視聴履歴の公開 | プライバシー |
| 視聴者への還元率公開 | クリエイター制度と混同 |

---

## 6. レベル制度（案）

| 要素 | 内容 |
| --- | --- |
| **XP 源** | 視聴時間 · コメント · ギフト · ストリーク |
| **表示** | レベル数字 + 称号（平易な日本語） |
| **特典** | コメント装飾 · バッジ · ランキング表示優先（軽微） |
| **上限** | 課金なしでも到達可能な cap を設ける |

---

## 7. ストリーク

| 種類 | 内容 |
| --- | --- |
| **ログインストリーク** | 日次訪問 |
| **視聴ストリーク** | 同一チャンネル連続視聴日 |
| **ライブ参加ストリーク** | ライブルーム滞在 |

**中断:** 1 日猶予（設計案）— 過度な FOMO を避ける。

---

## 8. ライブランキング

| 項目 | 内容 |
| --- | --- |
| **スコープ** | 配信ルーム内のみ（グローバルは Future） |
| **要素** | 応援 · ギフト · 視聴時間 · コメント（重み付けは Ops 調整） |
| **表示** | トップ N のみ · 匿名オプション |
