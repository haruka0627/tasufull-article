## 1. ミッション

### 1.1 一言

**「頑張ったクリエイターが一番報われるサービス」**

### 1.2 還元の考え方

**「利益を生み出したクリエイターには業界最高水準の還元」**

ただし **全員が常に高還元** ではない。

| 原則 | 内容 |
| --- | --- |
| **条件達成型** | 利益が成立する条件を達成したクリエイターのみ、高還元になる |
| **利益が先** | 利益が出るから還元する |
| **還元率先行禁止** | 還元率を先に決めてから運営するサービスではない |
| **持続可能性** | インフラコスト · 不正 · 低品質配信を吸収できない還元は行わない |

## 2. コアバリュー

| # | バリュー | 説明 |
| --- | --- | --- |
| 1 | **利益を最優先** | プラットフォームが赤字でクリエイター還元を続けられない構造は作らない |
| 2 | **利益を再投資** | 還元 · 新人支援 · イベント · インフラ改善へ循環 |
| 3 | **長期運営** | 短期の PV 競争より、継続可能なクリエイター経済 |
| 4 | **配信者ファースト** | 収益 · 成長 · ツール · 公平な評価 |
| 5 | **視聴者も楽しめる** | 応援 · コレクション · ランキング · ストリーク等で参加型体験 |
| 6 | **インフラコストを常に意識** | ライブ帯域 · エンコード · ストレージ · CDN を制度設計に組み込む |

## 3. プロダクト三本柱

```
通常動画（ロング）
    ↓
ショート動画
    ↓
ライブ配信
```

| 柱 | 役割 | 収益 |
| --- | --- | --- |
| **通常動画** | 資産蓄積 · 検索 · チャンネルブランド | 広告 · メンバーシップ · スーパーコメント（将来） |
| **ショート** | 新規流入 · 拡散 · ライブ/ロングへの導線 | 同上 · 流入効率重視 |
| **ライブ** | リアルタイム収益 · コミュニティ · イベント | **赤字にならない設計** · 条件達成型還元 |

詳細: [MONETIZATION.md](./MONETIZATION.md)
