## 1. 目的

ライブ配信を **赤字にしない** · **利益を作る** · **視聴者とクリエイター双方が楽しめる** 仕組みを定義する。

---

## 2. コアフロー（採用 ✅）

```
配信開始
    ↓
【第 1 フェーズ】30 分無料
    ↓
条件達成？ ──No──→ 配信終了（または有料延長のみ案内）
    │
   Yes
    ↓
【第 2 フェーズ】30 分延長（条件達成型）
    ↓
（任意）オーバーゲージ · 応援 · ランキング
    ↓
配信終了 → 録画 / ハイライト（将来）
```

### 2.1 延長条件（設計案 · 数値未確定）

| 条件例 | 目的 |
| --- | --- |
| 同時接続 · 視聴維持が閾値以上 | 無視聴延長防止 |
| 応援 / ギフトが infra コストを上回る | 赤字延長防止 |
| Creator Score · ランクが Standard 以上 | 新人の無制限延長防止 |
| ポリシー違反なし | 品質 |

**未達時:** 配信終了を **デフォルト**。延長は例外ではなく **達成型**。

## 3. 採用（✅ Adopted）

| 機能 | 概要 |
| --- | --- |
| **30 分無料 → 条件延長 30 分** | コアタイムボックス |
| **ライブゲージ** | 視聴 · 応援 · 時間で充填 — 延長条件の可視化 |
| **応援（Cheer）** | リアルタイムリアクション · ゲージ加算 |
| **ライブランキング** | ルーム内応援ランキング |
| **オーバーゲージ** | ゲージ上限突破 → ボーナス時間 / エフェクト（コスト上限付き） |
| **Raid** | 配信終了時に別チャンネルへ視聴者導線 |
| **ラジオ配信** | 低解像度 · 音声中心 — infra コスト削減モード |

## 6. 不採用 / 禁止

| 項目 | 理由 |
| --- | --- |
| 無条件無制限配信 | インフラ赤字 |
| 延長条件なしの 24h 配信 | 同上 |
| ボット視聴でゲージ操作 | 不正 · Score 減点 |
| ライブ中の自動 AI 返信 | 人間最終判断 |

## 7. ライブゲージ

```
[████████░░░░] 65%  → 延長条件まであと 35%
```

| 要素 | 内容 |
| --- | --- |
| **入力** | 同時接続 · 平均視聴時間 · 応援 · コメント |
| **出力** | 延長可否 · オーバーゲージ可否 |
| **表示** | クリエイター + 視聴者（シンプル UI） |
| **非公開** | 内部 weight · infra コスト試算 |

---

## 8. オーバーゲージ

| 項目 | 内容 |
| --- | --- |
| **トリガ** | ゲージ 100% 到達 + 応援継続 |
| **効果** | ボーナス時間（例: +5〜15 分 cap）· 視覚エフェクト |
| **上限** | 1 配信あたり max 延長時間 · infra cap |

---

## 9. Raid

| 項目 | 内容 |
| --- | --- |
| **タイミング** | 配信終了直前 |
| **動作** | 視聴者を指定チャンネルのライブ / 待機画面へ |
| **制限** | 相手クリエイターの opt-in · 1 日回数 cap |
